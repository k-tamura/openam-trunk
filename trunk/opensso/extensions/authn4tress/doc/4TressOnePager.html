Sun and ActivIdentity 4TRESS OTP Authentication Module as an OpenSSO Extension.  


The auth4TRESS OpenSSO extension is an OpenSSO authentication module that interfaces with the one-time password (OTP) methods supported by ActivIdentity 4TRESS Authentication Server. The integration of OpenSSO with 4TRESS Authentication Server allows enterprises to incorporate any OTP authentication method into the OpenSSO authentication framework

4TRESS OTP Authentication Module Features

1. Support of all OTP authentication methods supported by 4TRESS Authentication Server including:
  •      ActivIdentity SKI algorithm (as supported by the ActivIdentity/tokens)
  •      Open Authentication (OATH) based OTP (as supported by ActivIdentity DisplayCard and 3rd party token vendors)
  •      MasterCard's Chip Authentication Program (CAP) and Visa Dynamic Passcode Authentication (DPA) for EMV Authentication
  •      Static passwords (either as a primary authenticator or temporary login while waiting for a replacement device)
2. The Authentication module works with OpenSSO and has been tested against Sun Java System Access Manager 7.0 and 7.1

3. This module only works for 4TRESS Authentication Server 3.5 and above.

4. Version 1 features:
  •         Supports retrieving the outcome of an authentication request to a 4TRESS Authentication Server. This version does not maintain a 4TRESS Session Identifier.
  •         This module supports username/OTP credentials and not Challenge/Response OTP methods.
  •         4TRESS Authentication Server exposes an Apache Axis 1.2 web services for authentication. The authentication module uses this interface and needs to be installed with Apache Axis 1.2
  •         The authentication module supports localization, however only the .en_US properties files are currently distributed.
