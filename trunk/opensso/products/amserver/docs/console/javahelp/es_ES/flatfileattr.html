Atributos de repositorio de archivos sin formato

Los siguientes atributos se utilizan para configurar un repositorio de archivos sin formato:

Nombre de clase del complemento de repositorio de archivos

Este atributo especifica el archivo de clase de Java que proporciona la implementación para los archivos sin formato. Este atributo no debe modificarse.

Directorio del repositorio de archivos

Define el directorio base en el que se almacenan las identidades y sus atributos.

Caché

Si está habilitado (valor predeterminado), las identidades y sus atributos se almacenarán en la caché. Las siguientes solicitudes no tendrán acceso al sistema de archivos.

Tiempo para actualizar la caché

Si el almacenamiento en caché está habilitado, este atributo determina el intervalo de tiempo (en minutos) tras el que se comprueban las entradas de la caché para detectar los cambios realizados en el sistema de archivos. El mecanismo de comprobación se basa en marcas de tiempo.

Clases de objeto de usuarios de archivos

Define las clases de objeto que se agregan automáticamente a los usuarios durante su creación.

Atributo de contraseña

Proporciona el nombre de atributo que contiene la contraseña utilizada para la autenticación. Este atributo se utiliza para autenticar el usuario cuando se encuentra habilitado el módulo de autenticación de almacén de datos.

Atributo de estado

Proporciona el nombre de atributo que almacena el estado de la identidad. Los valores del atributo de estado son active o inactive. Se utiliza durante la autenticación de la identidad. Si una identidad es inactive, no se autenticará el usuario.

Atributos de hash

Proporciona una lista de los atributos cuyos valores tendrán un algoritmo hash y se almacenarán en los archivos. Una vez que tengan un algoritmo hash, no se pueden obtener los valores originales. Sólo es posible recuperar los valores con un algoritmo hash. Se utiliza para garantizar la privacidad cuando determinados atributos no deben almacenarse de forma permanente, pero se utilizan para la verificación. Un ejemplo de este tipo de atributo es el atributo de contraseña de la identidad.

Atributos cifrados

Proporciona una lista de los atributos cuyos valores se cifrarán y almacenarán en los archivos. Aunque se hayan cifrado y almacenado, al llamar a las API del repositorio de identidades, se devolverán los valores originales no cifrados De esta forma, se impide que los usuarios accedan directamente al sistema de archivos y lean atributos confidenciales.
