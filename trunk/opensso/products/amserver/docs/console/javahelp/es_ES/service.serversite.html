Servidores y sitios

Estos atributos permiten la administración centralizada de la configuración de los servidores y sitios de toda la implementación.

Varias (dos o más) instancias de OpenSSO pueden implementarse en, como mínimo, dos servidores host diferentes. Por ejemplo, puede implementar dos instancias en un servidor y una tercera en otro. También puede implementar todas las instancias en distintos servidores. Puede configurar también las instancias de OpenSSO en el modo de conmutación por error de sesión, si es necesario para la implementación.

Uno o varios equilibradores de carga enrutan las solicitudes del cliente a diversas instancias de OpenSSO. Puede configurar cada equilibrador de carga en función de los requisitos de implementación (por ejemplo, para una carga por turnos o un promedio de carga) con el fin de distribuir la carga entre las instancias de OpenSSO. Un equilibrador de carga simplifica la implementación y soluciona además determinados problemas como, por ejemplo, el servidor de seguridad entre el cliente y los servidores "back-end" de OpenSSO. Puede utilizar un equilibrador de carga de hardware o software con la implementación de OpenSSO. Todas las instancias de OpenSSO acceden a la misma instancia de Directory Server.
