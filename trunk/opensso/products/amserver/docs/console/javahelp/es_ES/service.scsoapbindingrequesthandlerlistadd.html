Para configurar un administrador de solicitudes

La interfaz RequestHandler debe implementarse en el servidor por parte de cada servicio de identidad basado en Liberty que utilice el Servicio de enlace de SOAP. El atributo “Lista de administradores de solicitudes” guarda información sobre las clases de implementación de los servicios Web que implementa el administrador de solicitudes. The Request Handler List displays entries that contain key/value pairs separated by a pipe as in: key=disco|class=com.example.identity.liberty.ws.disco.DiscoveryService .

  1. Haga clic en Nuevo para ver los atributos del nuevo administrador de solicitudes o haga clic en un valor de clave configurado para modificar los atributos existentes.
  2. Proporcione los valores de los atributos según la siguiente información:
    key

    El parámetro requerido es la última parte de la ruta del URI a un punto final de SOAP. El punto final de SOAP de OpenSSO es el servlet del receptor SOAP. El URI al receptor de SOAP es protocol ://host:port/ deploy-uri/Liberty/key. If you define disco as the key, the URI path to the SOAP endpoint for the corresponding Servicio de detección would be: protocol://host:port /amserver/Liberty/disco. Distintos clientes de servicios utilizan distintas claves al conectarse al receptor de SOAP.

    class

    El parámetro requerido especifica el nombre de la clase de implementación del administrador de solicitudes para un determinado servicio de identidades. For example: class=com.example.identity.liberty.ws.disco.DiscoveryService .

    Acción SOAP

    Este parámetro aporta un URI opcional específico de SOAP.

  3. Haga clic en Aceptar para completar la configuración del administrador de solicitudes.
  4. Haga clic en Guardar en la página del Servicio de enlace de SOAP para completar la configuración del servicio.
