Detección de cliente

Un paso inicial del proceso de autenticación consiste en identificar el tipo de cliente que realiza la solicitud HTTP(S). Esta función de OpenSSO se conoce como detección de cliente. La información de la dirección URL se utiliza para recuperar las características del cliente. En función de estas características, se mostrarán las páginas de autenticación adecuadas. Por ejemplo, cuando se utiliza el navegador Netscape para solicitar una página web, OpenSSO 8.0 muestra una página de inicio de sesión HTML. Una vez validado el usuario, el tipo de cliente (explorador Netscape) se agrega al token de sesión. Los atributos definidos en el servicio de detección de cliente son atributos globales.

Tipos de cliente

Para poder detectar los tipos de clientes, OpenSSO debe reconocer sus características de identificación. Estas características identifican las propiedades de todos los tipos admitidos en forma de datos del cliente. Este atributo permite modificar los datos de cliente mediante la interfaz de Client Manager. Para acceder a Client Manager, haga clic en el vínculo Editar. OpenSSO contiene los siguientes tipos de clientes de serie:

  • HDML

  • HTML

  • JHTML

  • VoiceX

  • WML

  • XHTML

  • cHTML

  • iHTML

For descriptions of these client types, see the http://docs.sun.com/app/docs/coll/1483.1.

Administrador de cliente

El Administrador de cliente es la interfaz que muestra los clientes base, los estilos y las propiedades asociadas, y permite agregar y configurar dispositivos. Los tipos de cliente base se muestran en la parte superior del Administrador de cliente. Estos tipos de cliente contienen las propiedades predeterminadas que pueden heredar todos los dispositivos que pertenezcan al tipo de cliente.

Tipo de cliente

El Administrador de cliente agrupa a todos los clientes disponibles, incluido el propio tipo de cliente base, en la lista de tipos de cliente. Puede modificar las propiedades de cada cliente haciendo clic en el nombre del dispositivo. A continuación, las propiedades se mostrarán en la ventana Client Editor (Editor de cliente). Para editar las propiedades, seleccione las siguientes clasificaciones en la lista desplegable:

Plataforma de hardware

Contiene las propiedades del hardware del dispositivo, como el tamaño de visualización, los conjuntos de caracteres admitidos, etc.

Plataforma de software

Contiene las propiedades del entorno de aplicación del dispositivo, el sistema operativo y el software instalado.

Características de red

Contiene las propiedades que describen el entorno de red, incluidos los portadores compatibles.

BrowserUA

Contiene los atributos relacionados con el agente de usuario del explorador que se ejecuta en el dispositivo.

WapCharacteristics

Contiene las propiedades del entorno del Protocolo de aplicaciones inalámbricas (WAP) admitido por el dispositivo.

PushCharacteristicNames

Contiene las propiedades del entorno WAP admitido por el dispositivo.

Propiedades adicionales

Contiene las propiedades del entorno del Protocolo de aplicaciones inalámbricas (WAP) admitido por el dispositivo.

Nota – Para obtener definiciones de propiedades específicas, consulte el Protocolo de aplicaciones inalámbricas Open Mobile Alliance Ltd. (OMA), versión del 20-Oct-2001.

Tipo de cliente predeterminado

Este atributo define el tipo de cliente predeterminado obtenido a partir de la lista de tipos de cliente en el atributo Client Types (Tipos de cliente). El valor predeterminado es genericHTML.

Clase de detección de cliente

Este atributo define la clase de detección del cliente para el que se enrutan todas las solicitudes de detección de clientes. La cadena que este atributo devuelve debe coincidir con uno de los tipos de cliente que se indican en el atributo Client Types (Tipos de cliente). La clase de detección de cliente predeterminada es com.sun.mobile.cdm.FEDIClientDetector. OpenSSO contiene también com.iplanet.services.cdm.ClientDetectionDefaultImpl .

Detección de cliente

Habilita la detección de clientes. Si se habilita la detección de clientes (valor predeterminado), cada solicitud se enruta mediante la clase especificada en el atributo Client Detection Class (Clase de detección de cliente). De forma predeterminada, la capacidad de detección de cliente está habilitada. Si no se selecciona este atributo, OpenSSO supone que el cliente es genericHTML y el acceso se obtendrá desde un navegador HTML.
