Para crear un nuevo encadenamiento de autenticación

  1. Haga clic en el nombre del dominio para el que desea agregar un nuevo encadenamiento de autenticación.
  2. Seleccione la ficha Autenticación.
  3. Haga clic en Nuevo en la lista Authentication Chaining (Cadena de autenticación).
  4. Introduzca un nombre para la cadena de autenticación.
  5. Haga clic en Aceptar.
  6. Haga clic en Agregar para definir la instancia del módulo de autenticación que desee incluir en la cadena. Para ello, seleccione el nombre de la instancia del módulo en la lista de instancias. Los nombres mostrados en esta lista se crean en el atributo de instancias del módulo.
  7. Seleccione los criterios para la cadena. Estos indicadores establecen criterios de obligación para el módulo de autenticación para el que están definidos. Existe una jerarquía para el cumplimiento de estos criterios. Required (Obligatorio) es el nivel superior y Optional (Opcional) es el inferior:
    Requisito

    Es necesaria la instancia del módulo para que el proceso se realice con éxito. Si es así, la autenticación continúa con los elementos inferiores de la lista de cadenas de autenticación. Si falla, se devuelve el control inmediatamente a la aplicación (la autenticación no continúa con los elementos inferiores de la lista de cadenas de autenticación).

    Obligatorio

    Es necesaria la autenticación de este módulo para que el proceso se realice con éxito. Si falla alguno de los módulos obligatorios, toda la cadena de autenticación fallará al final. Sin embargo, independientemente de que falle o no un módulo obligatorio, el proceso continuará con el siguiente módulo de la cadena.

    Suficiente

    No es necesaria la instancia del módulo para que el proceso se realice con éxito. Si se realiza con éxito, se devuelve el control inmediatamente a la aplicación (la autenticación no continúa con las siguientes instancias de módulos de la lista). Si falla, la autenticación continúa con los elementos inferiores de la lista de cadenas de autenticación.

    Opcional

    No es necesaria la instancia del módulo para que el proceso se realice con éxito. Independientemente de si falla o no, la autenticación continúa con los elementos inferiores de la lista de cadenas de autenticación.

  8. Introduzca las opciones para la cadena. Esto permite definir opciones adicionales para el par de clave y valor del módulo. Si hay varias opciones, éstas se separan mediante un espacio.
  9. Defina los siguientes atributos:
    URL de inicio de sesión correcta

    Especifica la dirección URL a la que se redirigirá el usuario si la autenticación se realiza con éxito.

    URL de inicio de sesión no válida

    Especifica la dirección URL a la que se redirigirá el usuario si la autenticación presenta errores.

    Authentication Post Processing Class

    Define el nombre de la clase de Java utilizado para personalizar los procesos posteriores a la autenticación para inicios de sesión válidos o erróneos.

  10. Haga clic en Guardar.
