Personalización de SAMLv2 XACML PDP

XACML PDP contiene los siguientes atributos que pueden personalizarse:

  • Enumeración de la compatibilidad del protocolo

  • Alias de la clave firmada

  • Alias de la clave de cifrado

  • Autorización básica

  • Consulta sobre la decisión de autorización firmada

  • Servicio de autorización

Enumeración de la compatibilidad del protocolo

Muestra la versión de XACML PDP compatible con este proveedor.

urn:liberty:iff:2003-08 hace referencia a Liberty Identity Federation Framework versión 1.2.

urn:liberty:iff:2002-12 hace referencia a Liberty Identity Federation Framework versión 1.1.

Alias de la clave firmada

Define el alias de clave que se utiliza para firmar solicitudes y respuestas.

Alias de la clave de cifrado

Define el alias de clave para el cifrado XACML.

Autorización básica

Para proteger los puntos finales SOAP puede habilitarse la autorización básica. Todo proveedor que acceda a estos puntos finales deberá tener el usuario y la contraseña definidas en las siguientes dos propiedades: Nombre de usuario y Contraseña.

Consulta sobre la decisión de autorización firmada

Cuando está habilitado, este atributo obliga que todas las consultas se firmen para la decisión de autorización XACML.

Servicio de autorización

Este atributo define el tipo (enlace) de la solicitud de autorización, así como el punto final de URL para recibir la solicitud. De forma predeterminada, el tipo de enlace es SOAP.
