Para crear o modificar un agente
  1. Haga clic en la ficha Agents (Agentes).
  2. Haga clic en Nuevo.
  3. Introduzca los valores para los siguientes campos:

    Name (Nombre). Introduzca el nombre o la identidad del agente. Se trata del nombre que utilizará el agente para iniciar una sesión en Federated Access Manager. No se permiten nombres de varios bytes.

    Password (Contraseña). Introduzca la contraseña del agente. Esta contraseña debe ser diferente a la utilizada por el agente durante la autenticación de LDAP.

    Confirmar contraseña . Confirme la contraseña.

    Estado de dispositivo. Introduzca el estado de dispositivo del agente. Si se define en Active (Activo), el agente podrá autenticarse en Federated Access Manager y establecer comunicación con esta aplicación. Si, por el contrario, se define en Inactive (Inactivo), el agente no podrá autenticarse en Federated Access Manager.

  4. Haga clic en Aceptar.
  5. Una vez creado el agente, puede editar también los siguientes campos:

    Description (Descripción). Escriba una breve descripción del agente. Por ejemplo, puede introducir el nombre de instancia del agente o el nombre de la aplicación que está protegiendo.

    Valor y clave de agente. Defina las propiedades con un par de clave/valor. Federated Access Manager utiliza esta propiedad para recibir las solicitudes de agente para las aserciones de credenciales relacionadas con los usuarios. Actualmente, sólo hay una propiedad válida, las restantes se omitirán. Use el siguiente formato:

    agentRootURL=http://server_name:port/
