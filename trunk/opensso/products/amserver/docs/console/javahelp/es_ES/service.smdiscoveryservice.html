Servicio de detección

El paso inicial para acceder a los datos de identidad es determinar dónde se encuentra la información. (Por ejemplo, el servicio de detección que guarda la información de la tarjeta de crédito de la entidad principal o el servidor que guarda el servicio de calendario de la entidad principal.) Normalmente, hay uno o más servicios en una red que permiten que otras entidades realicen una acción en los datos de identidad. Debido a que no se espera que los clientes mantengan un seguimiento de estos servicios o que sepan en cuál pueden confiar, es necesario disponer de un servicio de detección. Un servicio de detección es básicamente una interfaz de servicio Web para los recursos de detección. Un recurso de detección es un registro de las ofertas de recursos. Una oferta de recursos define una asociación entre una unidad de datos de identidad y la instancia de servicio que proporciona el acceso a dichos datos. Un identificador de recursos es un identificador de recursos exclusivo (URI, Unique Resource Identifier) registrado con el servicio de detección que señala a un determinado recurso de detección. Cuando un cliente envía una solicitud para obtener algunos tipos de datos, se incluye el identificador de recursos que el Servicio de detección de OpenSSO utiliza para localizar el proveedor de los servicios web de los atributos solicitados. El Servicio de detección muestra una oferta de recursos que contiene la información necesaria para localizar los datos.

Nota – Para poder disponer de acceso, el proveedor que aloja el Servicio de perfil personal de Liberty debe registrarse en el Servicio de detección en nombre de cada principal de identidad.
