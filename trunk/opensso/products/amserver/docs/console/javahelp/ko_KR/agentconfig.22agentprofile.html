2.2 정책 에이전트

OpenSSO는 이전 버전의 정책 에이전트 2.2와 호환됩니다. 정책 에이전트 2.2가 설치된 배포 컨테이너에서 로컬로 정책 에이전트 2.2를 구성해야 합니다. 따라서 OpenSSO 콘솔에서는 매우 한정된 수의 정책 에이전트 2.2 옵션만 구성할 수 있습니다.

비밀번호

비밀번호는 에이전트 프로필을 만들 때 설정됩니다. 그러나 이후에 언제든지 비밀번호를 변경할 수 있습니다.

비밀번호 확인

비밀번호 확인은 에이전트 프로필을 만들 때 수행됩니다. 비밀번호를 변경하면 변경된 비밀번호를 확인해야 합니다.

상태

에이전트를 만들면 활성 옵션이 선택됩니다. 에이전트가 제공하는 보호 기능을 제거하려는 경우에만 비활성을 선택하십시오.

설명

필요한 경우 추가할 수 있는 에이전트의 설명입니다.

에이전트 키 값

CDSSO를 사용하는 경우 쿠키 하이재킹을 방지하도록 배포를 구성할 때 필요한 설정입니다.

이 속성은 키와 값이 한 쌍을 이루어 하나의 키 역할을 합니다. 이 속성은 OpenSSO에서 사용자의 자격 증명 명제에 대한 에이전트 요청을 받는 데 사용됩니다. 이 키-값 쌍에는 하나의 속성만 유효합니다. 다른 모든 속성은 무시됩니다. 다음 형식을 사용합니다.

agentRootURL=protocol://hostname:port/

항목은 정확해야 합니다. 예를 들어 키를 나타내는 agentRootURL과 같은 문자열은 대소문자를 구분합니다.
