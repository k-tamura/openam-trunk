인증

사용자가 다른 인증 모듈 유형을 사용하여 로그인하기 전에 일반 인증 서비스를 영역에 대한 서비스로 등록해야 합니다. 핵심 인증 서비스를 사용하면 OpenSSO 관리자가 영역의 인증 매개 변수에 대한 기본값을 정의할 수 있습니다. 지정된 인증 모듈에 정의된 대체 값이 없는 경우 이러한 값을 사용할 수 있습니다. 핵심 인증 서비스의 기본값은 amAuth.xml 파일에 정의되며 설치 후 Directory Server에 저장됩니다.

인증 등록 정보는 다음과 같습니다.

고급 등록 정보

[고급 등록 정보] 버튼을 누르면 영역에 대해 정의할 수 있는 핵심 인증 속성이 표시됩니다. 전역 속성은 영역에 적용되지 않으므로 표시되지 않습니다.

기본 인증 체인

영역에 사용하는 인증 체인을 정의합니다. [인증 연결] 속성에서 새 체인을 만들면 풀다운 목록에 표시되어 선택할 수 있습니다. 자세한 내용은 새 인증 체인 만들기를 참조하십시오.

관리자 인증 체인

관리자에 대해서만 인증 체인을 정의합니다. 관리자의 인증 체인이 최종 사용자의 체인과 달라야 하는 경우 이 속성을 사용할 수 있습니다. 이 속성에 구성된 모듈은 OpenSSO 콘솔에 액세스할 때 상속됩니다.

기본 성공 URL

영역에 대한 인증 성공 시 사용자가 리디렉션되는 URL을 지정합니다.

모듈 인스턴스

영역에 대한 OpenSSO 기본 인증 모듈을 정의 및 구성할 수 있습니다. 같은 인증 모듈의 여러 인스턴스를 추가할 수 있습니다. 자세한 내용은 새 인증 모듈 인스턴스 만들기를 참조하십시오.

인증 연결

인증을 하나 이상 구성할 수 있으므로 사용자는 모든 인증에 인증 자격 증명을 전달해야 합니다. 이를 인증 연결이라고 합니다. 이 속성을 사용하여 영역에 대한 인증 연결 옵션을 정의할 수 있습니다. 만든 체인은 기본 인증 체인 목록에 표시됩니다. 자세한 내용은 새 인증 체인 만들기를 참조하십시오.
