J2EE 정책 에이전트

이 인터페이스를 사용하여 J2EE 에이전트 인스턴스를 구성할 수 있습니다. 여기서 설명한 등록 정보는 에이전트를 만드는 동안 중앙 집중화된 구성을 선택한 경우에만 적용됩니다. 로컬 구성을 선택한 경우 이 에이전트와 관련된 등록 정보를 해당 에이전트 설치 디렉토리의 OpenSSOAgentConfiguration.properites 파일에서 편집해야 합니다.

사용 가능한 구성 유형은 다음 범주로 구분됩니다.

  • 전역

  • 응용 프로그램

  • SSO

  • OpenSSO 서비스

  • 기타

  • 고급

전역

이러한 등록 정보는 에이전트가 보호하는 모든 응용 프로그램에 적용됩니다.

그룹

해당되는 경우 드롭다운 목록에서 그룹을 선택하여 이 에이전트를 해당 그룹에 할당합니다. 이 목록은 이전에 구성된 그룹으로 구성됩니다.

비밀번호

비밀번호는 에이전트 프로필을 만들 때 설정됩니다. 그러나 이후에 언제든지 비밀번호를 변경할 수 있습니다.

비밀번호 확인

비밀번호 확인은 에이전트 프로필을 만들 때 수행됩니다. 비밀번호를 변경하면 변경된 비밀번호를 확인해야 합니다.

상태

에이전트를 만들면 활성 옵션이 선택됩니다. 에이전트가 제공하는 보호 기능을 제거하려는 경우에만 비활성을 선택하십시오.

에이전트 알림 URL

에이전트가 알림 수신자를 등록하는 데 사용하는 URL입니다.

에이전트 구성 저장소 위치

필요한 경우 구성 위치를 중앙 집중됨 또는 로컬 옵션 중에서 사용 가능한 옵션으로 변경합니다. 중앙 집중화된 위치를 사용하면 콘솔 등과 같이 중앙 집중화된 방식으로 구성을 제어할 수 있습니다.

로컬 옵션은 이전 버전과의 호환을 위해 제공됩니다. 로컬 구성 옵션을 선택한 경우 에이전트가 해당 에이전트 설치 디렉토리의 OpenSSOAgentConfiguration.properites 파일에 있는 로컬 구성을 사용합니다. 또한 콘솔에는비밀번호, 비밀번호(확인) 및 상태 등록 정보만 표시됩니다.

구성 재로드 간격

구성 재로드 사이의 간격(초)입니다. 이 등록 정보를 0으로 설정하면 핫스왑 메커니즘이 비활성화됩니다.

에이전트 구성 변경 알림

사용 설정되면 구성이 변경되는 경우 에이전트가 OpenSSO 서버로부터 알림 메시지를 받을 수 있습니다.

에이전트 필터 모드

필터의 작업 모드입니다. 이 등록 정보의 경우 전역 값을 특정 필터 모드가 없는 모든 응용 프로그램에 적용하도록 설정할 수 있습니다.

[맵 키]에는 웹 응용 프로그램 이름을 입력하고 [해당 맵 값]에는 적절한 값을 입력합니다. 유효한 값은 ALL, J2EE_POLICY, URL_POLICY, SSO_ONLY 및 NONE입니다. 전역 필터 모드로 ALL을 설정하려면[맵 키] 필드를 비워두고 [해당 맵 값] 필드에 ALL을 입력하십시오.

자원 액세스 거부 URI

사용자 정의 액세스 거부 페이지의 URL입니다. 값을 지정하지 않으면 에이전트가 HTTP 403(금지) 상태를 반환합니다.

HTTP 세션 바인딩

사용 설정되면 로그인 실패 시 HTTP 세션이 무효화됩니다. 사용자에게 SSO 세션이 없거나 기본 사용자 이름이 SSO 사용자 이름과 일치하지 않습니다.

GOTO 매개 변수 이름

goto 매개 변수 이름을 나타내는 문자열이며, 에이전트에서 사용자를 적절한 인증 서비스로 리디렉션하는 데 사용됩니다. 이 매개 변수의 값은 인증 서비스에서 사용자를 원래 요청한 대상으로 리디렉션하는 데 사용됩니다.

로그인 시도 한계

사용자 요청이 차단될 때가지 단일 브라우저 세션에서 허용되는 실패한 로그인 시도 횟수입니다. 이 값이 0이면 이 기능이 비활성화됩니다

사용자 정의 응답 헤더

사용자 정의 헤더의 매핑 목록입니다. 헤더는 클라이언트 브라우저에서 에이전트가 설정합니다. [맵 키]에는 헤더 이름을 입력하고 [해당 맵 값]에는 헤더 값을 입력합니다.

리디렉션 시도 한계

사용자 요청이 차단될 때까지 단일 브라우저 세션에서 허용되는 연속 단일 지점 리디렉션 횟수입니다. 이 값이 0이면 이 기능이 비활성화됩니다

에이전트 디버그 수준

로깅되는 디버그 메시지 유형입니다. 이 설정은 에이전트 호스트에 로컬로 저장되는 디버그 로그의 수준을 결정합니다.

사용자 매핑 모드

에이전트가 사용자 아이디를 결정하는 데 사용하는 메커니즘입니다.

사용자 속성 이름

사용자 아이디가 포함되는 속성의 이름입니다. 사용자 매핑 모드 등록 정보가 USER_ID로 설정된 경우에는 이 등록 정보의 값을 사용하지 않습니다.

사용자 기본 플래그

사용 설정되면 사용자 아이디만 단독으로 사용하는 것이 아니라 인증된 사용자의 기본을 사용하여 사용자를 인증합니다. 사용자 매핑 모드 등록 정보가 USER_ID로 설정된 경우 이 등록 정보의 값이 사용됩니다.

사용자 토큰 이름

세션에서 인증된 사용자의 사용자 아이디에 대한 세션 등록 정보 이름입니다. 이 등록 정보의 값은 사용자 매핑 모드 등록 정보가 USER_ID로 설정되고 사용자 기본 플래그 등록 정보를 사용하지 않는 경우에 사용됩니다.

감사 액세스 유형

사용자 URL 액세스 시도에 따라 에이전트가 로깅하는 메시지 유형입니다.

감사 로그 위치

감사 메시지가 로깅되는 위치입니다.

원격 로그 파일 이름

OpenSSO에 대한 각 URL 액세스가 기록되는 OpenSSO 로그 파일의 이름입니다.

로컬 감사 로그 회전

사용 설정되면 로컬 감사 로그 회전 크기 등록 정보에서 지정한 크기에 도달하는 경우 감사 로그 파일이 회전됩니다.

로컬 감사 로그 회전 크기

로컬 감사 로그 파일이 새 파일로 회전되는 크기입니다.

FQDN 확인

사용 설정되면 FQDN 기본값과 FQDN 맵 값을 확인합니다.

FQDN 기본값

사용자가 자원에 액세스하기 위해 사용하는 정규화된 호스트 이름입니다.

FQDN 가상 호스트 맵

실제 또는 유효한 호스트 이름에 대한 매핑입니다. 이 매핑은 인식되지 않는 호스트 이름을 인식되는 호스트 이름으로 매핑합니다. 이 매핑은 사용자가 IP 주소 또는 잘못된 URL을 입력했거나 보호되는 자원에 대해 가상 호스트를 사용하는 경우 유용합니다.

[맵 키] 및 [해당 맵 값]의 값을 입력하고 [추가]를 누릅니다.

응용 프로그램

이러한 등록 정보는 응용 프로그램 전용으로 사용됩니다.

로그인 양식 URI

응용 프로그램의 web.xml form-login-page 요소에 해당하는 절대 URI 목록입니다.

로그인 오류 URI

응용 프로그램의 web.xml form-error-page 요소에 해당하는 절대 URI 목록입니다.

내부 로그인 사용

사용 설정되면 에이전트가 로그인 컨텐트 파일 이름 등록 정보에서 지정한 사용자 정의 내부 컨텐트를 사용합니다. 이 속성을 사용하지 않으면 에이전트가 기본 내부 컨텐트를 사용합니다.

로그인 컨텐트 파일 이름

사용자 정의 로그인 컨텐트 파일의 전체 경로와 이름입니다.

응용 프로그램 로그아웃 처리기

응용 프로그램 전용 매핑이며, 각각 로그아웃 처리에 사용할 처리기를 식별하는 데 사용됩니다.

[맵 키] 필드에는 로그아웃 처리기를 입력하고 [해당 맵 값] 필드에는 응용 프로그램 로그아웃 처리기 클래스 이름을 입력합니다. [추가]를 누릅니다.

응용 프로그램 로그아웃 URI

응용 프로그램 전용 매핑이며, 각각 요청 URI를 식별하는 데 사용됩니다. 지정된 요청 URI가 있는 경우에는 로그아웃 이벤트를 나타냅니다.

[맵 키] 필드에는 로그아웃 URI를 입력하고 [해당 맵 값] 필드에는 응용 프로그램 로그아웃 URI를 입력합니다. [추가]를 누릅니다.

로그아웃 요청 매개 변수

응용 프로그램 전용 매핑이며, 각각 매개 변수를 식별하는 데 사용됩니다. HTTP 요청에 지정된 매개 변수가 있는 경우에는 로그아웃 이벤트를 나타냅니다.

로그아웃 Introspect 사용

사용 설정되면 에이전트가 HTTP 요청 본문을 검색하여 로그아웃 매개 변수를 찾습니다.

로그아웃 시작 URI

응용 프로그램 전용 매핑이며, 각각 성공적인 로그아웃 후 그리고 해당하는 경우 후속 인증 성공 후 시작점으로 사용할 URI를 나타냅니다.

[맵 키] 필드에는 해당 값을 입력하고 [해당 맵 값] 필드에는 로그아웃 시작 URI를 입력합니다. [추가]를 누릅니다.

강제 적용되지 않는 URI

인증을 수행하지 않는 URI 목록입니다. 이 목록의 모든 URI에 대해 에이전트에서 보호를 수행하지 않으며 자격 증명도 요청하지 않습니다.

강제 적용되지 않는 URI 반전

사용 설정되면 강제 적용되지 않는 URI 목록이 강제 적용되는 목록으로 바뀝니다. 따라서 강제 적용되지 않는 URI 목록 등록 정보에 값으로 나열된 모든 URI가 적용되며 다른 URI는 적용되지 않습니다.

강제 적용되지 않는 URI 캐시 사용

사용 설정되면 평가 결과가 강제 적용되지 않는 URI에 대해 캐시됩니다. 따라서 평가 결과가 강제 실행되지 않는 URI 등록 정보와 연결된 값 목록에 대해 캐시됩니다.

강제 적용되지 않는 URI 캐시 크기

강제 적용되지 않는 URI 캐시 사용 등록 정보를 사용하는 경우 이 등록 정보에 지정된 값을 사용하여 캐시 크기를 설정합니다.

강제 적용되지 않는 클라이언트 IP 목록

인증이나 권한 부여를 거치지 않는 요청의 클라이언트 IP 주소 목록입니다. 이 목록의 모든 IP 주소에 대해 액세스 권한이 부여되지만 인증 또는 권한 부여를 위한 자격 증명을 요청하지는 않습니다.

강제 적용되지 않는 IP 반전 목록

사용 설정되면 강제 적용되지 않는 클라이언트 IP 목록이 강제 적용되는 목록으로 바뀝니다. 따라서 강제 적용되지 않는 클라이언트 IP 목록 등록 정보에 값으로 나열된 모든 IP 주소가 적용되며 다른 IP 주소는 적용되지 않습니다.

강제 적용되지 않는 IP 캐시 플래그

사용 설정되면 평가 결과가 강제 적용되지 않는 IP 주소에 대해 캐시됩니다. 따라서 평가 결과가 강제 적용되지 않는 클라이언트 IP 목록 등록 정보와 연결된 값 목록에 대해 캐시됩니다.

강제 적용되지 않는 IP 캐시 크기

강제 적용되지 않는 IP 캐시 플래그 등록 정보를 사용하는 경우 이 등록 정보에 지정된 값을 사용하여 캐시 크기를 설정합니다.

프로필 속성 불러오기 모드

이 모드에서는 요청에서 사용할 추가 사용자 프로필 속성을 불러올 수 있습니다.

프로필 속성 매핑

프로필 속성 이름을 HTTP 헤더 이름과 매핑하기 위한 목록입니다. HTTP 헤더 이름은 현재 인증된 사용자의 특정 이름 아래에 채워집니다.

응답 속성 불러오기 모드

이 모드에서는 요청에 사용할 추가 사용자 응답 속성을 불러올 수 있습니다.

응답 속성 매핑

응답 속성 이름을 HTTP 헤더 이름과 매핑하기 위한 목록입니다. HTTP 헤더 이름은 현재 인증된 사용자의 특정 이름 아래에 채워집니다.

쿠키 구분 문자

동일한 속성을 가진 여러 값을 하나의 쿠기로 설정하는 경우 각 값을 구분하는 데 사용하는 문자입니다.

불러오기 속성 날짜 형식

속성을 HTTP 헤더로 설정하는 경우 사용할 날짜 속성 값의 형식입니다. 이 형식은 java.text.SimpleDateFormat을 기준으로 합니다.

속성 쿠키 인코딩

사용 설정되면 이 속성의 값은 쿠키로 설정하기 전에 URL 인코딩됩니다.

세션 속성 불러오기 모드

이 모드에서는 요청에 사용할 추가 사용자 세션 속성을 불러올 수 있습니다.

세션 속성 매핑

세션 속성 이름을 HTTP 헤더 이름과 매핑하기 위한 목록입니다. 세션 속성 이름은 현재 인증된 사용자의 특정 이름 아래에 채워집니다.

기본 권한 속성

인증된 사용자에 대한 권한 속성 목록입니다. 이러한 속성은 유효한 OpenSSO 세션을 갖는 모든 사용자에게 권한이 부여됩니다.

권한 있는 속성 유형

각 사용자에 대해 불러올 권한 있는 속성 유형의 목록입니다.

권한 있는 속성을 소문자로

권한이 있는 속성 유형을 true 또는 false 값과 매핑한 목록입니다. true는 속성 유형을 소문자로 변환하며, 변환하지 않을 경우에는 false로 설정합니다.

권한 있는 세션 속성
