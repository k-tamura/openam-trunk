Magasins de données

Un magasin de données est une base de données dans laquelle vous pouvez stocker des attributs utilisateur et des données de configuration utilisateur.

Access Manager fournit un plug-in de référentiel d'identités qui se connecte à une structure de référentiel d'identités LDAPv3. Ce nouveau modèle vous permet de consulter et de récupérer les informations utilisateur Access Manager sans apporter de modification à votre base de données utilisateur existante. La structure Access Manager intègre les données du plug-in de référentiel d'identités avec les données d'autres plug-ins Access Manager afin de constituer une identité virtuelle pour chaque utilisateur. Access Manager peut alors utiliser cette identité universelle dans les processus d'authentification et d'autorisation au sein de plusieurs référentiels d'identités. L'identité d'utilisateur virtuelle est détruite lorsque la session de l'utilisateur prend fin.

Pour plus d'informations sur la création d'un magasin de données et sur la configuration du plug-in de référentiel, reportez-vous à Création d'un magasin de données LDAPv3.

Si vous avez installé Access Manager en mode Domaines, vous pouvez configurer le SDK d'Access Manager pour gérer les données Directory Server. Reportez-vous à Création d'un plug-in de référentiel Access Manager.

Création d'un magasin de données LDAPv3

  1. Sélectionnez le domaine auquel vous souhaitez ajouter un nouveau magasin de données.

  2. Cliquez sur l'onglet Magasin de données.

  3. Cliquez sur Nouveau dans la liste des magasins de données.

  4. Entrez le nom du magasin de données.

  5. Définissez les attributs du plug-in de référentiel LDAPv3. Pour une description des attributs, reportez-vous à Attributs du plug-in du référentiel LDAPv3.

  6. Cliquez sur Terminer.
