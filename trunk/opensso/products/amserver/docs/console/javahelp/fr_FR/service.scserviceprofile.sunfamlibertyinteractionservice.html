Service d'interaction Liberty

Le fournisseur de services Web redirige l'utilisateur pour l'interaction

Cet attribut indique si le fournisseur de services Web redirige ou non l'utilisateur pour obtenir son autorisation. La valeur par défaut est yes.

Le fournisseur de services Web redirige l'utilisateur pour l'interaction en vue des données

Ceci lance une interaction pour obtenir l'autorisation de l'utilisateur ou collecter des données supplémentaires. Cette propriété indique si le fournisseur de services Web redirige ou non l'utilisateur pour collecter d'autres données. La valeur par défaut est yes.

Durée d'interaction prévue par le fournisseur de services Web

Cet attribut indique la durée (en secondes) prévue par le fournisseur de services Web pour effectuer une interaction et restituer le contrôle au client du service Web. Par exemple, le fournisseur de services Web reçoit une demande indiquant que le client du service Web attendra un maximum de 30 secondes (définies dans Durée d'interaction prévue par le client du service Web) pour l'interaction. Si cet attribut est défini sur 40 secondes, le fournisseur de services Web renvoie une erreur SOAP (timeNotSufficient ), indiquant que la durée est insuffisante pour l'interaction.

Le fournisseur de services Web oblige à ce que returnToURL soit SSL

Cet attribut indique si le fournisseur de services Web applique ou non un paramètre returnToURL HTTPS spécifié par le client du service Web. Les spécifications Liberty Alliance Project stipulent que la valeur de cette propriété est toujours yes. La valeur false est essentiellement utilisée pour un déploiement par phases aisé.

Le fournisseur de services Web oblige à ce que returnToHost soit identique à requestHost

Cet attribut indique si le fournisseur de services Web applique ou non les valeurs d'adresse de returnToHost et requestHost si elles sont identiques. Les spécifications Liberty Alliance Project stipulent que la valeur de cette propriété est toujours yes. La valeur false est essentiellement utilisée pour un déploiement par phases aisé.

Emplacement de feuille de style HTML

Cet attribut pointe sur l'emplacement de la feuille de style utilisée pour afficher la page d'interaction au format HTML.

Emplacement de feuille de style WML

Cet attribut pointe sur l'emplacement de la feuille de style utilisée pour afficher la page d'interaction au format WML.

URL d'interaction du fournisseur de services Web

Cet attribut spécifie l'URL où le servlet WSPRedirectHandler est déployé. Le servlet traite la partie fournisseur de services des interactions pour la redirection de l'utilisateur.

URL d'interaction du fournisseur de services Web si derrière un équilibreur de charge

Définit l'URL du gestionnaire de redirection du fournisseur de services Web affichée par un équilibreur de charge.

Liste des URL d'interaction du cluster (site) du fournisseur de services Web derrière l'équilibreur de charge

Définit les URL du gestionnaire de redirection du fournisseur de services Web des serveurs de confiance dans le cluster.

Classe de configuration d'interaction

Cet attribut spécifie la classe qui fournit les méthodes d'accès permettant de lire les configurations d'interaction.

Options de participation du client du service Web à l'interaction

Cet attribut indique le niveau d'interaction selon lequel le client du service Web participe s'il est configuré pour participer à la redirection de l'utilisateur. Les valeurs possibles sont interactIfNeeded, doNotInteract et doNotInteractForData. La valeur affirmative interactIfNeeded est la valeur par défaut.

Le client du service Web inclut userInteractionHeader

Cet attribut indique si le client du service Web inclut ou non un en-tête SOAP pour spécifier certaines préférences d'interaction en fonction des spécifications Liberty. La valeur par défaut est yes.

Le client du service Web redirige l'utilisateur pour l'interaction

Cet attribut spécifie si le client du service Web participe ou non à la redirection de l'utilisateur. La valeur par défaut est yes.

Durée d'interaction prévue par le client du service Web

Cet attribut définit le temps maximal (en secondes) pendant lequel le client du service Web attend que le fournisseur de services Web effectue sa partie de l'interaction. Le fournisseur de services Web ne lance pas d'interaction s'il est probable que l'interaction dure plus de temps que la durée définie. Par exemple, le fournisseur de services Web reçoit une demande dans laquelle cette propriété est définie sur un maximum de 30 secondes. Si la propriété Durée d'interaction prévue par le client du service Web du fournisseur de services Web est définie sur 40 secondes, le fournisseur de services Web renvoie une erreur SOAP (timeNotSufficient), indiquant que la durée est insuffisante pour l'interaction.

Le client du service Web oblige à ce que l'URL de redirection soit SSL

Cet attribut spécifie si le client du service Web applique ou non HTTPS dans les URL de redirection. Les spécifications Liberty Alliance Project stipulent que la valeur de cette propriété est toujours yes, ce qui indique que le fournisseur de services Web ne redirige pas l'utilisateur lorsque la valeur de redirectURL (spécifiée par le fournisseur de services Web) n'est pas une URL HTTPS. La valeur false est essentiellement utilisée pour un déploiement par phases aisé.
