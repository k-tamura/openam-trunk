Configuration d'un identificateur de site

Cet attribut définit tout site hébergé par le serveur sur lequel OpenSSO est installé. Une valeur par défaut et un ID de site généré automatiquement sont définis pour l'hôte lors de l'installation. Il peut y avoir plusieurs entrées. Par exemple, l'équilibrage de charge ou plusieurs instances d'OpenSSO partageant le même Directory Server doivent être tous définis. Le point de départ est l'attribut Identificateurs de site dans l'écran SAML sous l'interface Fédération. Les ID de site sont définis dans l'écran de configuration Serveurs et sites. Pour plus d'informations, reportez-vous à la rubrique Serveurs et Sites.

  1. Cliquez sur Nouveau pour ajouter un identificateur de site ou cliquez sur le nom d'un identificateur de site configuré pour modifier son profil.

    Les attributs de l'identificateur de site s'affichent.

  2. Fournissez les valeurs pour les attributs d'identificateur de site en vous basant sur les informations suivantes :
    ID d'instance

    La valeur de cette propriété est protocol ://host: port.

    En cas de configuration SAML pour SSL (dans les sites source et de destination), vérifiez que le protocole défini ici est https//.

    ID de site

    L'ID de site est un identificateur généré pour chaque site (bien que la valeur puisse être identique pour plusieurs serveurs placés derrière un équilibreur de charge). Vous pouvez utiliser une classe du package com.sun.identity.saml.common afin de générer manuellement cet identificateur si besoin est. Tapez la commande suivante sur la ligne de commandes :

    % java -classpath FM-classpath com.sun.identity.saml.common.SAMLSiteID protocol://host:port
    Nom de l'émetteur

    La valeur par défaut de cette propriété est host :port, mais ce peut être un URI quelconque.

  3. Cliquez sur OK pour terminer la configuration de l'identificateur de site.
  4. Cliquez sur Enregistrer dans la page Propriétés du site local pour finaliser la configuration SAML.
