Service Web d'authentification

Le Service Web d'authentification définit le mode d'authentification avec SOAP. L'échange des informations d'authentification entre un consommateur de services Web (WSC) et le fournisseur de services Web (WSP) est réalisé à l'aide de messages de liaison SOAP. Ces messages forment une série de demandes du client et de réponses du serveur propre au mécanisme SASL (Simple Authentication and Security Layer) défini (ou mode d'authentification). Lorsqu'il reçoit une demande d'authentification (ou une réponse du WSC), le WSP peut exiger d'autres informations, ou indiquer la réussite ou l'échec de l'authentification. Le Service Web d'authentification est permet d'authentifier un service auprès d'un autre service (et non un utilisateur auprès d'un autre utilisateur). Les étapes suivantes décrivent en détail la séquence d'actions entre le WSC et le Service Web d'authentification (un WSP).

  1. L'échange d'authentification débute par l'envoi par le WSC d'une demande d'authentification SASL au Service Web d'authentification pour le compte d'une identité principale.

    Le message de demande contient un identificateur pour l'entité principale et propose un ou plusieurs mécanismes SASL que le service peut choisir.

  2. Le Service Web d'authentification répond en définissant la méthode à utiliser et, le cas échéant, en exigeant d'autres informations.

    Si le Service Web d'authentification ne prend pas en charge les méthodes proposées, il répond en mettant fin à l'échange.

  3. Le WSC répond en fournissant les données d'identité requises pour la méthode d'authentification choisie.

  4. Le Service Web d'authentification répond en approuvant ou en désapprouvant l'authentification.

    Si la demande est approuvée, la réponse inclut les informations d'authentification dont le WSC a besoin pour appeler d'autres services Web (comme le Service de découverte).

Les attributs généraux suivants du Service Web d'authentification peuvent être configurés pour votre implémentation.

  • Liste des gestionnaires de mécanismes

  • Question/réponse intervalle de nettoyage — Cet attribut spécifie l'intervalle de nettoyage (en secondes) dans la classe d'implémentation de gestionnaire de mécanismes CRAM-MD5 com.sun.identity.liberty.ws.authnsvc.mechanism.CramMD5MechanismHandler . Le thread d'intervalle commence à nettoyer la carte de questions/réponses en fonction de cette valeur.

  • Classes de transformation — Cet attribut spécifie le nom de transformation au mappage de classe d'implémentation. Les valeurs sont séparées par une virgule avec "|" comme délimiteur pour le nom de transformation et le nom de classe d'implémentation. Par exemple, nom1|classe1, nom2|classe2.
