Pour créer un nouveau groupe

Si vous le souhaitez, effectuez cette tâche dans la console d'administration de Federated Access Manager . Cette tâche s'applique aux groupes d'agents Web et J2EE.

Créez un groupe si vous souhaitez que les agents héritent de propriétés spécifiques du groupe. Les agents J2EE peuvent hériter des propriétés d'un groupe d'agents J2EE et les agents Web des propriétés d'un groupe d'agents Web.

  1. Cliquez sur Agents dans l'onglet Configuration.
  2. Si nécessaire, sélectionnez l'onglet correspondant au type d'agent approprié.
    • Agents Web :

      L'onglet Web est destiné aux agents Web et est sélectionné par défaut. Aucune autre sélection n'est nécessaire.

    • Agents J2EE :

      L'onglet J2EE est destiné aux agents J2EE. Le cas échéant, cliquez sur cet onglet.

  3. Dans la section Groupe, cliquez sur Nouveau.

    Dans le champ Nom, entrez le nom du nouveau groupe.

  4. Dans le champ URL de serveur, entrez l'URL du serveur de Federated Access Manager.
  5. Cliquez sur Créer.

    La console crée le groupe d'agents et affiche la page des agents (Web ou J2EE) avec un lien vers le groupe.

    Pour effectuer d'autres tâches de configuration du groupe, cliquez sur ce lien afin d'afficher la page Modification de groupe d'agents.

    Les attributs que vous pouvez définir pour configurer un groupe sont les mêmes que ceux correspondant à un agent individuel, à la différence que les attributs Groupe, Mot de passe et Confirmation de mot de passe ne sont pas disponibles au niveau du groupe.

    Remarque – En outre, sachez que certains attributs de groupes sont déjà affectés à des valeurs de variables qui, dans la plupart des cas, ne doivent pas être changées. Voici un exemple d'une de ces valeurs :

    @AGENT_PROTO@://@AGENT_HOST@:@AGENT_PORT@/amagent
