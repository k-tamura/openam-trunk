Personnalisation d'une autorité d'authentification SAMLv2

L'autorité d'authentification SAMLv2 contient les attributs personnalisables suivants :

  • Signature et codage

  • Service de requête d'autorisation

  • Demande d'AssertionID

  • Alias cert

Signature et codage
Taille de la clé

Longueur des clés utilisées par l'entité Autorité d'attributs lors de son interaction avec une autre entité.

Algorithme

Algorithme de chiffrement utilisé pour interagir avec une autre entité.

Service de requête d'autorisation

Cet attribut définit l'URL auquel les demandes d'authentification sont envoyées.

Demande d'AssertionID

Définit les URL auxquels les ID d'assertion sont envoyés par un client à un fournisseur d'identités pour récupérer l'assertion correspondante. Location spécifie l'URL du fournisseur auquel la demande est envoyée. Les types de demande d'ID d'assertion sont les suivants :

  • SOAP

  • URI

Alias cert

Cet attribut définit les éléments de l'alias du certificat pour le fournisseur. signing spécifie l'alias du certificat du fournisseur, utilisé pour trouver le certificat de signature correct dans le keystore. Encryption spécifie l'alias du certificat du fournisseur, utilisé pour trouver le certificat de chiffrement correct dans le keystore.
