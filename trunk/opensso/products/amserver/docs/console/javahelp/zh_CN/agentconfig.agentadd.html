创建新代理

创建新代理也称为创建新的代理配置文件。您可以在 OpenSSO 管理控制台中执行该任务。某些单独步骤仅适用于 J2EE 和 Web 策略代理类型。

  1. 单击“访问控制”选项卡。
  2. 单击代理所属的领域的名称。
  3. 单击“代理”选项卡。
  4. 选择适当代理类型的选项卡。
    • Web 代理
    • J2EE 代理
    • Web 服务提供者代理
    • Web 服务客户机代理
    • STS 客户机代理

      “安全令牌服务”客户机代理具有两种可选类型：搜索代理和 STS 代理。有关这些代理类型的详细信息，参见STS 客户机。

    • 2.2 代理

      “2.2 代理”选项卡适用于“策略代理 2.2”软件集中的代理。

    • 代理验证程序
  5. 在“代理”部分中单击“新建”。
  6. 在“名称”字段中输入新代理配置文件的名称。
  7. 输入“密码”并确认。

    注意 – 此密码必须与您运行 agentadmin 程序以安装代理时在指定的代理配置文件密码文件中输入的密码相同。

    步骤 8 至 10 仅适用于 Web 代理和 J2EE 策略代理。

  8. 选择适当的配置：“本地”或“集中”。

    选择本地配置时，无法从控制台编辑与该代理相关的属性。在此情况下，代理从代理安装目录中的本地文件 OpenSSOAgentBootstrap.properties 和 OpenSSOAgentConfiguration.properites 中检索配置信息。对于本地配置的代理，可直接编辑 OpenSSOAgentConfiguration.properites 文件来更改属性值。

  9. 在“服务器 URL”字段中，输入 OpenSSO 服务器 URL。

    例如：

    http://OpenSSO_EnterpriseHost.example.com:8080/OpenSSO
  10. 在“代理 URL”字段中，输入代理应用程序 agentapp 的 URL。

    例如：

    Web 代理

    http://agentHost.example.com:8090

    J2EE 代理

    http://agentHost.example.com:8090/agentapp

  11. 单击“创建”。

    控制台创建代理配置文件，并再次显示含有到新代理配置文件的链接的代理页面。

    要对代理配置文件进行其他配置，单击该链接显示“编辑”代理页面。
