配置請求處理程式

每一個使用 SOAP 連結服務並基於 Liberty 的識別服務都必須在伺服器端實作 RequestHandler 介面。請求處理程式清單屬性會儲存 Web 服務 (用來執行請求處理程式) 實作類別的相關資訊。請求處理程式清單會顯示包含以管道分隔的鍵/值對的項目，如下所示：key=disco|class=com.example.identity.liberty.ws.disco.DiscoveryService。

  1. 按一下 [新建] 以顯示 [新建請求處理程式] 屬性，或按一下已配置的鍵值以修改現有屬性。
  2. 根據下列資訊提供屬性值：
    key

    此必需的參數是 SOAP 端點之 URI 路徑的最後部份。Access Manager 中的 SOAP 端點是 SOAP 收件者 servlet。SOAP 收件者的 URI 是 protocol ://host:port/ deploy-uri/Liberty/key。如果您將 disco 定義為 key，則對應探索服務的 SOAP 端點的 URI 路徑將是：protocol://host:port /amserver/Liberty/disco。連線至 SOAP 收件者時，不同服務用戶端使用不同的 key。

    class

    此必需的參數指定特殊識別服務的請求處理程式實作類別的名稱。例如：class=com.example.identity.liberty.ws.disco.DiscoveryService。

  3. 按一下 [確定] 以完成請求處理程式的配置。
  4. 按一下SOAP 連結服務頁面上的 [儲存] 以完成服務配置。
