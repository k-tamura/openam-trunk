SAMLv2 SOAP 連結

SAMLv2 SOAP 連結服務可以為 OpenSSO 用戶端和 OpenSSO 伺服器之間提供基於 SOAP 的 SAMLv2 請求和回應訊息交換。收到的請求會被委託給請求處理器進行進一步的處理。請求處理程式與中介別名的關鍵字位於 SOAP 連結服務 URL 中。中介別名與 RequestHandler 的對映儲存在 SAMLv2 SOAP 連結服務中，您可以從 OpenSSO 配置存放區讀取。

請求處理程式清單

RequestHandlerList 是關鍵字/值組項目的清單，其中包含中介別名與 RequestHandler 實作的對映。如果要配置 OpenSSO 8.0 伺服器用做策略決定點 (Policy Decision Point, PDP)，則必須設定此屬性。

關鍵字 是策略決定點中介別名，而 類別 是 Java 類別名稱，此類別是可以處理 XACML 請求之 RequestHandler 介面的實作。

例如，如果 XACML 策略決定點的中介別名為 /pdp，而介面的實作為 com.sun.identity.xacml.plugins.XACMLAuthzDecisionQueryHandler，則關鍵字應設定為 /pdp，而類別應設定為 com.sun.identity.xacml.plugins.XACMLAuthzDecisionQueryHandler。
