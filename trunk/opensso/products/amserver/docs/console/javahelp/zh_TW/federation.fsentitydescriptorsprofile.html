修改提供者實體或是附屬提供者實體的一般屬性

在建立實體之後，您可以編輯其設定檔。這可能包括修改屬性、新增識別提供者及服務提供者，或配置附屬提供者。編輯實體設定檔的一般屬性可能需要修改已經定義的共用屬性、新增連絡人員或是提供選擇性的組織資訊。下列程序用於編輯一般屬性。起點為聯合之下的實體畫面。

  1. 選取配置的實體名稱以修改其設定檔。

    依預設，會顯示實體設定檔的 [一般] 屬性頁面。如果不在 [一般] 屬性頁面上，您可以從 [檢視] 功能表選取 [一般]。

  2. 編輯任何 [實體的一般屬性] 的值。
    實體名稱

    此屬性的靜態值為您建立實體時提供的名稱。

    類型

    此屬性的靜態值是實體的類型 (提供者或附屬提供者)。

    描述

    此屬性值為您建立實體時提供的說明。您可以修改原先輸入的說明，或在說明不存在時新增說明。

  3. 提供 [實體連絡人員設定檔] 屬性的值。

    [實體連絡人員設定檔] 屬性提供有關實體要連絡何人的相關資訊。這些是選用屬性。

    名字

    鍵入實體連絡人員的指定名字。

    姓氏

    鍵入實體連絡人員的姓氏。

    類型

    從下拉式功能表中選擇連絡類型：

    • 管理

    • 帳單

    • 技術

    • 其他

    公司

    鍵入此人所屬公司的名稱。

    Liberty 首要 ID

    鍵入統一資源識別碼 (URI)，它必須指向連絡人員的個人資訊設定檔的實例。

    電子郵件

    鍵入連絡人員的一個或多個電子郵件地址，並按一下 [新增]。

    電話號碼

    鍵入連絡人員的一個或多個電話號碼，並按一下 [新增]。

  4. 提供 [組織設定檔] 屬性的值。

    [組織設定檔] 屬性提供與主體互動時可能需要的基本資訊。這些是選用屬性。

    備註 – 如果 [名稱] 屬性包含值，則必須為 [顯示名稱] 及 [URL] 屬性新增值。

    名稱

    輸入實體組織的合法名稱，並按一下 [新增]。定義值的格式為 locale| organization-name。例如，en| example.com。

    顯示名稱

    輸入適合於向主體顯示的實體組織名稱，並按一下 [新增]。定義值的格式為 locale|organization-display-name。例如，en|example.com。

    URL

    輸入組織的 URL，並按一下 [新增]。定義值的格式為 locale| organization-URL。例如，en| http://www.example.com。

  5. 按一下 [儲存] 來完成配置，或繼續定義識別提供者、服務提供者或附屬提供者屬性的值。

    如果繼續進行配置，請參閱：

    • 新增識別提供者至提供者實體，以定義識別提供者屬性的值。
    • 新增服務提供者至提供者實體，以定義 [服務供應程式] 屬性的值。
    • 新增成員至附屬提供者實體，將成員寫入附屬提供者實體。
