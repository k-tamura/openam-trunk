建立新群組

如有需要，可在 Federated Access Manager 管理主控台中執行這項作業。這項作業適用於 Web 代理程式群組和 J2EE 代理程式群組。

如果您要讓代理程式繼承群組的特定特性，則請建立群組。J2EE 代理程式可繼承 J2EE 代理程式群組的特性，而 Web 代理程式可繼承 Web 代理程式群組的特性。

  1. 按一下 [配置] 標籤下的 [代理程式]。
  2. 如有必要，請選取適當代理程式類型的標籤。
    • Web 代理程式：

      [Web] 標籤用於 Web 代理程式，且為預設選項。不需要再進行選取。

    • J2EE 代理程式：

      [J2EE] 標籤用於 J2EE 代理程式。如果適用，請按一下此標籤。

  3. 在 [群組] 區段中，按一下 [新建]。

    在 [名稱] 欄位中，輸入新群組的名稱。

  4. 在 [伺服器 URL] 欄位中，輸入 Federated Access Manager 伺服器 URL。
  5. 按一下 [建立]。

    主控台會建立代理程式群組，並且再次顯示代理程式 (WebAgent 或 J2EEAgent) 頁面，其中含有群組的連結。

    若要進行額外的群組配置，請按一下此連結，以顯示 [編輯] 代理程式群組頁面。

    您可設定用來配置群組的屬性與用於個別代理程式的屬性相同，僅有的不同之處是 [群組]、[密碼] 和 [密碼確認] 屬性無法在群組層級使用。

    備註 – 同時，務必注意某些群組屬性已指定了變數值，而且在大部分情況下不應該變更。以下為這類值的範例：

    @AGENT_PROTO@://@AGENT_HOST@:@AGENT_PORT@/amagent
