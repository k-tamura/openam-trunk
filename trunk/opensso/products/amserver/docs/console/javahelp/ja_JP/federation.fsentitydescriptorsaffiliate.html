アフィリエイトエンティティーにメンバーを追加する

アフィリエイトエンティティーの作成後に、そのアフィリエイトにメンバーを追加できます。連携インタフェースの「エンティティー」から作業を始めます。

注 – この画面から定義済みの「共通属性」を変更することもできます。

  1. プロファイルを変更する設定済みアフィリエイトエンティティーの名前をクリックします。

    エンティティーのプロファイルページが表示されます。

  2. 「表示」メニューから「アフィリエイト」を選択します。
  3. 利用可能な任意のエンティティーを選択して、アフィリエイトに割り当てます。
  4. 「共通属性」の値を変更します。

    「共通属性」の値は、アイデンティティープロバイダ自体の一般的な定義です。

    名前

    適切な場合は、アフィリエイトの新規 URI を入力します。この属性の現在の値は、エンティティーを作成したときに「アフィリエイト名」属性に割り当てた URI です。新規の値は、アフィリエイト名の間で一意でなければなりません。

    所有者

    新規 URI を入力して、アフィリエイトに別の所有者または親オペレータを定義します。この属性の現在の値は、エンティティーを作成したときに「アフィリエイト所有者」属性に割り当てた URI です。この属性の新規の値は、新しい所有者/プロバイダが属するエンティティーの「エンティティー名」属性に定義された値と一致する URI です。新規の値は、アフィリエイト名の間で一意でなければなりません。

    注 – プロバイダを「アフィリエイト所有者」として定義しても、そのプロバイダがアフィリエイトのメンバーとして自動的に組み込まれることはありません。所有者がメンバーでもある場合は、URI をここでは所有者として定義し、エンティティー内部ではメンバーとして定義する必要があります。

    署名鍵: 鍵のエイリアス

    証明書は Java キーストアファイルに格納されます。ファイル内の各証明書はエイリアスにマップされ、証明書をフェッチするときにはそのエイリアスが使用されます。要求や応答の署名に使用する証明書に関連付けられているキーエイリアスを入力します。

    暗号化鍵: 鍵のエイリアス

    証明書は Java キーストアファイルに格納されます。ファイル内の各証明書はエイリアスにマップされ、証明書をフェッチするときにはそのエイリアスが使用されます。プロバイダのセキュリティー証明書に関連付けられているキーエイリアスを入力します。

    暗号化鍵: キーサイズ

    Web サービスコンシューマが別のエンティティーと対話するときに使用するキーの長さを入力します。

    暗号化鍵: 暗号化方式

    暗号化方式設定を指す URI を入力します。

  5. 「保存」をクリックします。
