サービス記述を設定する

サービス記述属性は、個別のプロトコルエンドポイントで実行中の Web サービスを定義します。これは、「ブートストラップのリソースオファリング」を設定する際に定義されます。サービスインスタンスに関する情報は、各種のコンテキスト間でやり取りされる必要があります。たとえば、定義されているディスカバリサービスは、リソースオファリング (それぞれにはサービスインスタンス記述が含まれる) の列挙を提供するアイデンティティーサービスの 1 つです。

  1. 「新規記述」をクリックして新規のサービス記述属性を表示するか、「編集」をクリックして既存の記述を変更します。
  2. 次の情報に基づいて属性の値を入力します。
    セキュリティーメカニズム ID

    この属性では、ディスカバリサービスがサポートする認証方法が追加されます。このセキュリティーメカニズムは、Web サービスコンシューマが Web サービスプロバイダに対して認証を受ける方法、またはメッセージレベルのセキュリティーを提供する方法を参照します。デフォルトでは、サービスインスタンスがサポートするすべての方法が選択されます。Web サービスコンシューマが認証方法を 1 つも選択せずに、その方法で要求を送信しようとすると、その要求は拒否されます。「サポートされている認証メカニズム」を参照してください。

    エンドポイント URL

    この属性の値は、SOAP-over-HTTP エンドポイントの URL です。URI スキーマは、https://soap.profile-provider.com/soap のように HTTP または HTTPS にする必要があります。

    SOAP アクション

    この属性は省略可能でその値は、サービスの具体的な WSDL ベースの記述にある wsdlsoap:operation 要素の wsdlsoap:soapAction 属性と等価です。

  3. 「了解」をクリックしてサービスの設定を完了します。
  4. ディスカバリサービスページで「保存」をクリックして、サービスの設定を完了します。
