フラットファイルリポジトリの属性

フラットファイルリポジトリの設定に使用される属性は次のとおりです。

ファイルリポジトリプラグインのクラス名

フラットファイルの実装を提供する Java クラスファイルを指定します。この属性は変更しないでください。

ファイルリポジトリディレクトリ

アイデンティティーとその属性が格納されるベースディレクトリを定義します。

キャッシュ

有効にすると (デフォルト)、アイデンティティーとその属性がキャッシュされます。後続の要求では、ファイルシステムにアクセスしません。

キャッシュを更新する時間

キャッシュを有効にした場合に、キャッシュ内のエントリがファイルシステム側で変更されたかどうかを調べる時間間隔を分単位で指定します。このチェックメカニズムはタイムスタンプに基づいています。

ファイルユーザーオブジェクトクラス

作成時に自動的にユーザーに追加されるオブジェクトクラスを定義します。

パスワード属性

認証に使用されるパスワードを含む属性名を指定します。この属性は、「データストア」認証モジュールが有効になっているときに、ユーザーを認証するのに使用されます。

状態属性

アイデンティティーの状態を格納する属性名を指定します。状態属性の値は、active または inactive のいずれかです。これは、アイデンティティーの認証中に使用されます。アイデンティティーが inactive の場合、ユーザーは認証されません。

ハッシュ属性

値がハッシュされてファイルに保存される属性のリストを指定します。一度ハッシュされると、元の値を取得することはできません。ハッシュ済みの値のみが取得されます。これは、検証に使用するが永久に保存することは適切でない属性が関係する機密性の保証に使用されます。アイデンティティーのパスワード属性は、このタイプの属性の例です。

暗号化された属性

値が暗号化されてファイルに保存される属性のリストを指定します。これらの属性は暗号化されて保存されますが、アイデンティティーリポジトリ API を呼び出すと元の値が復号化されて返されます。これにより、ユーザーがファイルシステムに直接アクセスして機密属性を読み取ることを防止できます。
