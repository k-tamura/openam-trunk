SafeWord

このモジュールにより、ユーザーが Secure Computing の SafeWord または SafeWord PremierAccess 認証サーバーを使用して認証できるようになります。SafeWord 認証属性はレルム属性です。属性は次のとおりです。

  • 「サーバー」

  • 「サーバー検証ファイルのディレクトリ」

  • 「ログを有効」

  • 「ログレベル」

  • 「ログファイル」

  • 「認証接続タイムアウト」

  • 「クライアントタイプ」

  • 「EASSP バージョン」

  • 「オーセンティケータ最小強度」

  • 「認証レベル」

サーバー

SafeWord サーバーまたは SafeWord PremiereAccess サーバーの名前とポートを指定します。ポート 7482 は SafeWord サーバーのデフォルトとして設定されています。SafeWord PremierAccess サーバーのデフォルトポート番号は 5030 です。

サーバー検証ファイルのディレクトリ

SafeWord クライアントライブラリがその検証ファイルを置くディレクトリを指定します。デフォルトは次のとおりです。

/var/opt/SUNWam/auth/safeword/serverVerification

このフィールドに異なるディレクトリを指定する場合は、SafeWord 認証を試みる前にそのディレクトリが存在する必要があります。

ログを有効

SafeWord ログを有効にします。デフォルトでは、SafeWord ログは有効になっています。

ログレベル

SafeWord ログレベルを指定します。ドロップダウンメニューでレベルを選択します。「デバッグ」、「エラー」、「情報」、および「なし」を選択できます。

ログファイル

SafeWord クライアントログのディレクトリパスとログファイル名を指定します。デフォルトのパスは /var/opt/SUNWam/auth/safeword/safe.log です。

異なるパスまたはファイル名を指定する場合は、SafeWord 認証を試みる前にそれらが存在している必要があります。SafeWord 認証に複数のレルムが設定され別々の SafeWord サーバーが使用されている場合、別々のパスを指定する必要があります。そうしないと、SafeWord 認証が行われる最初のレルムだけが認証されます。同様に、レルムが SafeWord サーバーを変更した場合、新しく設定された SafeWord サーバーの認証が行われる前に、指定されたディレクトリの swec.dat ファイルを削除する必要があります。

認証接続タイムアウト

SafeWord クライアント (OpenSSO) と SafeWord サーバーとの間のタイムアウト期間を秒単位で定義します。デフォルトは 120 秒です。

クライアントタイプ

モバイルクライアント、VPN、固定パスワード、チャレンジ/レスポンスなど、異なるクライアントとの通信に SafeWord サーバーが使用するクライアントタイプを定義します。

EASSP バージョン

この属性には、EASSP (Extended Authentication and Single Sign-on Protocol) のバージョンを指定します。このフィールドには、標準 (101)、SSL 暗号化 Premier Access (200)、Premier Access (201) のいずれかのプロトコルバージョンを指定できます。

オーセンティケータ最小強度

クライアントおよび SafeWord サーバー認証のオーセンティケータ最小強度を定義します。オーセンティケータ値はクライアントタイプごとに異なり、値が高いほどオーセンティケータ強度も強くなります。20 がもっとも高い値です。0 がもっとも低い値です。

認証レベル

認証レベルは認証方法ごとに個別に設定します。この値は、認証メカニズムの信頼度を示します。ユーザーが認証を受けると、この値がセッションの SSO トークンに格納されます。ユーザーがアクセスするアプリケーションに SSO トークンが提供されると、そのアプリケーションは格納されている値を使用して、ユーザーにアクセスを許可するのに十分なレベルかどうかを判別します。SSO トークンに格納されている認証レベルが必要な最小値に満たない場合、アプリケーションはユーザーにより高い認証レベルのサービスで認証を再度受けるよう要求することがあります。デフォルト値は 0 です。

注 – 認証レベルの指定がない場合、SSO トークンはコア認証属性の 「デフォルト認証レベル」で指定した値を格納します。
