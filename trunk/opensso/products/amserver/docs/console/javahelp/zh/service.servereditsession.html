会话

您可以通过会话属性配置其他站点和服务器实例的会话信息。

会话限制

以下属性设置服务器会话限制：

最大会话数

默认值为 5000。指定允许并发会话的最大数目。如果并发会话的最大数目超过此数目，则登录时就会发送“最大会话数”错误。

使会话无效的最长时间

默认值为 10。指定在会话创建后没有用户登录的情况下，要从会话表中删除无效会话所间隔的分钟数。该值应该始终大于“验证”模块属性文件中的超时值。

会话清除延迟

默认值为 60。指定延迟清除会话操作的时间（以分钟为单位）。会话超时后，会话将在这个延长时限内继续驻留在会话服务器中。客户机应用程序使用该属性通过 SSO API 检查会话是否超时。在延长时限即将结束时，会话将被销毁。如果用户注销或 OpenSSO 组件明确销毁会话，则在延长时限内不会维持会话。会话在这一延长时限内处于“无效”状态。

统计信息

以下属性设置统计信息配置：

记录的时间间隔

默认值为 60。指定统计信息记录之间的时间间隔（以分钟为单位）。最小为 5 秒，以避免 CPU 饱和。OpenSSO 将小于 5 秒的值视为 5 秒。

状态

默认值为 file。指定统计信息记录的位置。可能的值为：

  • 关 — 不记录统计信息。

  • 文件 — 将统计信息写入到指定目录下的文件中。

  • 控制台 — 将统计信息写入到 Web 服务器的日志文件中。

目录

在安装期间设置该值。示例：OpenSSO-base/server-URI/stats。指定将在其中创建调试文件的目录。

启用主机查找

默认值为“假”。在会话记录期间启用或禁用主机查找。

通知

以下属性设置通知配置：

通知池大小

默认值为 10。通过指定线程总数来定义池的大小。

通知线程池阈值

默认值为 100。指定最大任务队列长度。收到通知任务后，会被发送到任务队列以进行处理。如果队列达到最大长度，继续传入的请求将被拒绝，并会发出 ThreadPoolException，直到队列有空缺为止。

验证

以下属性设置验证配置：

不区分大小写的客户机 DN 比较

默认值为“真”。比较代理 DN。如果值为假，则比较会区分大小写。
