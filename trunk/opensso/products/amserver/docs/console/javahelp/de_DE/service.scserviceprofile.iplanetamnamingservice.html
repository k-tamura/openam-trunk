Namen

Der Namensdienst wird zum Abrufen und Festlegen der URLs, Plugins und Konfigurationen sowie für Anforderungsbenachrichtigungen für verschiedene OpenSSO-Dienste verwendet, z. B. für Sitzung, Authentifizierung, Protokollierung, SAML und Verbindung.

Dieser Dienst ermöglicht es den Clients, den korrekten Dienst-URL zu finden, wenn OpenSSO auf der Plattform mehrfach ausgeführt wird. Wenn eine Namens-URL gefunden wurde, entschlüsselt der Namensdienst die Benutzersitzungsinformationen und ersetzt dynamisch die Werte für Protokoll, Host und Anschluss mit den Parametern der Sitzung. Damit wird sichergestellt, dass der an den Dienst zurückgegebene URL auf den Host verweist, auf dem die Benutzersitzung erstellt wurde. Es existieren folgende Namensattribute:

  • URL des Profildienstes

  • URL des Sitzungsdienstes

  • URL des Protokollierungsdienstes

  • URL des Richtliniendienstes

  • URL des Authentifizierungsdienstes

  • URL des SAML-Webprofil/-Produktdienstes

  • URL des SAML-SOAP-Dienstes

  • URL des SAML-Webprofil-/POST-Dienstes

  • URL des SAML-Assertion-Manager-Dienstes

  • Dienst-URL für Verbindungassertions-Manager

  • URL für den Sicherheits-Token-Manager

  • URL für JAXRPC-Endpunkt

URL des Profildienstes

In diesem Feld ist ein Wert im folgenden Format zulässig:

%protocol://%host:%port/Server_DEPLOY_URI/profileservice

Diese Syntax ermöglicht das dynamische Ersetzen der Profil-URL, basierend auf den sitzungsspezifischen Parametern.
