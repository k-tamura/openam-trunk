Plattform

Mithilfe des Plattformdienstes können zusätzliche Server zur OpenSSO-Konfiguration hinzugefügt und andere Optionen auf der obersten Ebene der OpenSSO-Anwendung übernommen werden. Bei den Plattformdienstattributen handelt es sich um globale Attribute. Es existieren folgende Attribute:

  • Plattform-Ländereinstellung

  • Cookie-Domänen

  • Verfügbare Ländereinstellungen

  • Client-Zeichensätze

Plattform-Ländereinstellung

Der Wert dieses Attributs ist der Standard-Sprachenuntertyp, mit dem OpenSSO installiert wurde. Die Authentifizierungs-, Protokoll- und Verwaltungsdienste werden in der mit diesem Attribut eingestellten Sprache verwaltet. Der Standardwert ist en_US. Eine Liste der unterstützten Sprachuntertypen finden Sie unter Unterstützte Sprach- bzw. Ländereinstellungen.

Cookie-Domänen

Dieses Attribut gibt die Liste der Domänen an, die im Cookie-Header zurückgegeben werden, wenn während der Authentifizierung ein Cookie für den Browser des Benutzers eingerichtet wird. Falls dieses Attribut leer ist, wird keine Cookie-Domäne eingerichtet. Mit anderen Worten: Das OpenSSO-Sitzungscookie wird lediglich an OpenSSO weitergeleitet und an keinen anderen Server der Domäne.

Wenn für andere Server in der Domäne SSO erforderlich ist, muss in diesem Attribut die Cookie-Domäne angegeben werden. Wenn auf einem OpenSSO zwei Schnittstellen in unterschiedlichen Domänen vorhanden sind, müssen in diesem Attribut beide Cookie-Domänen angegeben werden. Wird ein Lastenausgleichssystem eingesetzt, muss die Domäne dieses Systems als Cookie-Domäne angegeben werden und nicht die Domäne der Server hinter diesem System. Der Standardwert für dieses Feld ist die Domäne des installierten OpenSSO.

Verfügbare Ländereinstellungen

Dieses Attribut speichert alle für die Plattform verfügbaren Ländereinstellungen. Verwenden Sie nach Möglichkeit eine Anwendung, die dem Benutzer die Auswahl der Ländereinstellungen ermöglicht. Eine solche Anwendung ruft dieses Attribut aus dem Plattformprofil ab und zeigt dem Benutzer die Liste der verfügbaren Ländereinstellungen an. Die vom Benutzer ausgewählte Ländereinstellung wird von der Anwendung im Benutzereintrag preferredLocale gespeichert.

Client-Zeichensätze

Dieses Attribut gibt die Zeichensätze für unterschiedliche Clients auf Plattformebene an. Es enthält eine Liste der Client-Typen mit den zugehörigen Zeichensätzen. Weitere Informationen finden Sie unter So erstellen Sie einen neuen Zeichensatz.
