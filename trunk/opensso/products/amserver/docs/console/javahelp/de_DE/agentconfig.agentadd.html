So erstellen Sie einen neuen Agenten

Das Erstellen eines neuen Agenten wird auch als Erstellen eines neuen Agentenprofils bezeichnet. Sie können diese Aufgabe an der OpenSSO-Administrationskonsole ausführen. Einige der einzelnen Schritte gelten nur für die Richtlinien-Agenten-Typen "J2EE" und "Web".

  1. Klicken Sie auf die Registerkarte "Zugriffssteuerung".
  2. Klicken Sie auf den Namen des Bereichs, zu dem der Agent gehören soll.
  3. Klicken Sie auf die Registerkarte "Agenten".
  4. Wählen Sie die Registerkarte für den entsprechenden Agententyp aus.
    • Web-Agenten
    • J2EE-Agenten
    • Webdienstanbieter-Agenten
    • Webdienst-Client-Agenten
    • STS Client-Agent

      Der Client-Agent für den Sicherheitstokendienst stellt zwei Typen zur Auswahl: Discovery-Agent und STS-Agent. Weitere Informationen zu den Agententypen finden Sie unter STS-Client.

    • 2.2-Agenten

      Die Registerkarte "2.2-Agenten" bezieht sich auf Agenten im Policy Agent 2.2-Software-Set.

    • Agenten-Authentifizierer
  5. Klicken Sie im Abschnitt "Agent" auf "Neu".
  6. Geben Sie im Feld "Name" den Namen des neuen Agentenprofils ein.
  7. Geben Sie das Passwort ein, und bestätigen Sie es.

    Achtung – Dieses Passwort muss mit dem Passwort in der Datei für das Agentenprofilpasswort übereinstimmen, die Sie angeben, wenn Sie den Agenten durch Ausführung des Programms agentadmin installieren.

    Die Schritte 8–10 gelten nur für Web- und J2EE-Richtlinienagenten.

  8. Wählen Sie die entsprechende Konfiguration: "Lokal" oder "Zentralisiert".

    Bei Auswahl der lokalen Konfiguration können die zu diesem Agenten gehörenden Eigenschaften nicht über die Konsole bearbeitet werden. In einem solchen Szenario erhält der Agent Konfigurationsinformationen aus den lokalen Dateien, OpenSSOAgentBootstrap.properties und OpenSSOAgentConfiguration.properites , im Verzeichnis der Agenteninstallation. Bei einem lokal konfigurierten Agenten ändern Sie die Eigenschaftswerte durch direkte Bearbeitung der Datei OpenSSOAgentConfiguration.properites .

  9. Geben Sie im Feld "Server-URL" die URL des OpenSSO-Servers ein.

    Beispiel:

    http://OpenSSO_EnterpriseHost.example.com:8080/OpenSSO
  10. Geben Sie im Feld "Agenten-URL" die URL der Agentenanwendung, agentapp, ein.

    Beispiel:

    Web-Agenten

    http://agentHost.example.com:8090

    J2EE-Agenten

    http://agentHost.example.com:8090/agentapp

  11. Klicken Sie auf "Erstellen".

    Die Konsole erstellt das Agentenprofil und zeigt die Agentenseite mit einer Verknüpfung zum neuen Agentenprofil an.

    Wenn Sie weitere Konfigurationen am Agentenprofil vornehmen möchten, klicken Sie auf diese Verknüpfung, um die Seite zur Bearbeitung des Agenten anzuzeigen.
