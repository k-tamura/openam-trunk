So fügen Sie einer Anbieter-Entity einen Identity-Anbieter hinzu

Der erste Schritt beim Hinzufügen eines Identity-Anbieters zu einer Anbieter-Entity besteht darin, die Art des Identity-Anbieters zu definieren, der hinzugefügt wird.

  • Ein Host-Anbieter befindet sich auf demselben Server wie Access Manager.

  • Ein Remote-Anbieter befindet sich nicht auf demselben Server wie Access Manager.

Das folgende Verfahren beschreibt diesen ersten Schritt. Der Ausgangspunkt ist der Bildschirm "Entities" unter "Verbindung".

  1. Wählen Sie den Namen einer konfigurierten Anbieter-Entity, um deren Profil zu ändern.

    Die Profilseite der Entity wird angezeigt.

  2. Wählen Sie im Menü "Ansicht" die Option "Identity-Anbieter" aus.
  3. Wählen Sie den hinzuzufügenden Providertyp aus.

    • Wählen Sie "Neuer Host-Anbieter", wenn sich der Anbieter auf dem gleichen Server wie Access Manager befindet.

    • Wählen Sie "Neuer Remote-Anbieter", wenn sich der Anbieter nicht auf dem gleichen Server wie Access Manager befindet.

    Die entsprechende Profilseite für Identity-Anbieter wird angezeigt.

  4. Wählen Sie abhängig von Ihrer Auswahl im vorangegangenen Schritt eines der folgenden Verfahren aus:
    • So fügen Sie einer Anbieter-Entity einen neuen Host-Identity-Anbieter hinzu
    • So fügen Sie einer Anbieter-Entity einen neuen Remote-Identity-Anbieter hinzu
