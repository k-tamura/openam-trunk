配置網站識別碼

此屬性定義由安裝 OpenSSO 之伺服器代管的任何網站。安裝期間會為主機定義預設值和自動產生的網站 ID。可以有多個項目。例如，負載平衡或共用同一 Directory Server 的多個 OpenSSO 實例全都需要加以定義。起點是聯合介面 SAML 畫面上的 [網站識別碼] 屬性。網站 ID 是在 [伺服器與網站配置] 畫面中定義的。如需詳細資訊，請參閱伺服器與網站。

  1. 按一下 [新建] 以新增一個新的網站識別碼，或是在已配置的網站識別碼的名稱上按一下以修改其設定檔。

    會顯示 [網站識別碼] 屬性。

  2. 根據下列資訊提供網站識別碼屬性的值：
    實例 ID

    此特性的值為 protocol://host:port。

    如果為 SSL 配置 SAML (同時在資源與目標網站內)，請確保此處定義的協定為 https//。

    網站 ID

    網站 ID 是為每個網站 (雖然在負載平衡器之後的多個伺服器之值將會相同) 所產生的識別碼。如果需要的話，com.sun.identity.saml.common 套裝軟體內有一個類別，可用來手動產生此識別碼。在指令行鍵入如下：

    % java -classpath FM-classpath com.sun.identity.saml.common.SAMLSiteID protocol://host:port
    發行者名稱

    此特性的預設值為 host:port，但它可以是任何 URI。

  3. 按一下 [確定] 以完成網站識別碼配置。
  4. 按一下 [本機網站特性] 頁面上的 [儲存] 以完成 SAML 配置。
