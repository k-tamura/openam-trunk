探索服務

存取識別資料的初始步驟是判定資訊所在的位置。(例如，哪一個識別服務保留主體的信用卡資訊，或哪一個伺服器儲存主體的行事曆服務)。通常，網路上有一或多個服務允許其他實體對識別資料執行動作。因為並未預期用戶端追蹤這些服務，或知道可以信任哪些服務，所以它們需要探索服務。探索服務實質上是探索資源的 Web 服務介面。探索資源是資源提供的登錄。資源提供定義識別資料片段與提供該資料存取權的服務實例之間的關聯。資源識別碼是在探索服務中註冊的唯一資源識別碼 (unique resource identifier, URI)，它指向特定探索資源。當用戶端傳送對某些類型資料的請求時，它會包括 OpenSSO 探索服務用來為尋找請求的屬性之 Web 服務提供者的資源識別碼。探索服務會傳回資源提供，其中包含尋找資料所需的資訊。

備註 – 為了進行存取，Liberty 個人設定檔服務 的代管提供者需要代表每一個識別主體註冊 探索服務。

  • 支援的指令

  • 探索查找的策略評估

  • 探索更新的策略評估

  • 授權程式外掛程式類別：

  • 項目處理程式外掛程式類別：

  • ResourceID 對映程式外掛程式類別

  • 認證回應訊息

  • 啟動程式的階段作業環境敘述

  • 為啟動程式加密階段作業環境中的名稱識別碼

  • 隱含資源

  • 啟動程式的資源提供
