建立新代理程式

建立新代理程式也稱為建立新代理程式設定檔。您可以在 OpenSSO 管理主控台中執行這項作業。有些單獨的步驟只適用於 J2EE 和 Web 策略代理程式類型。

  1. 按一下 [存取控制] 標籤。
  2. 按一下代理程式所屬的範圍的名稱。
  3. 按一下 [代理程式] 標籤。
  4. 選取適當代理程式類型的標籤。
    • Web 代理程式
    • J2EE 代理程式
    • Web 服務提供者代理程式
    • Web 服務用戶端代理程式
    • STS 用戶端代理程式

      「安全性記號服務」用戶端代理程式有兩種類型可讓您選擇，即探索代理程式和 STS 代理程式。如需這些代理程式類型的詳細資訊，請參閱STS 用戶端。

    • 2.2 代理程式：

      [2.2 代理程式] 標籤用於「策略代理程式 2.2」軟體集中的代理程式。

    • 代理程式認證程式
  5. 在 [代理程式] 區段中，按一下 [新建]。
  6. 在 [名稱] 欄位中，輸入新代理程式設定檔的名稱。
  7. 輸入並確認 [密碼]。

    注意 – 此密碼必須與您執行 agentadmin 程式安裝代理程式時，在指定的代理程式設定檔密碼檔中輸入的密碼相同。

    步驟 8 到 10 只適用於 Web 和 J2EE 策略代理程式。

  8. 選取適當的配置：「本機」或「集中」。

    選取本機配置時，無法從主控台編輯與此代理程式相關的特性。在這類情況下，代理程式會從代理程式安裝目錄中的本機檔案 OpenSSOAgentBootstrap.properties 和 OpenSSOAgentConfiguration.properites 擷取配置資訊。若為本機配置的代理程式，可透過直接編輯 OpenSSOAgentConfiguration.properites 檔案來變更特性值。

  9. 在 [伺服器 URL] 欄位中，輸入 OpenSSO 伺服器 URL。

    例如：

    http://OpenSSO_EnterpriseHost.example.com:8080/OpenSSO
  10. 在 [代理程式 URL] 欄位中，輸入代理程式應用程式 agentapp 的 URL。

    例如：

    Web 代理程式

    http://agentHost.example.com:8090

    J2EE 代理程式

    http://agentHost.example.com:8090/agentapp

  11. 按一下 [建立]。

    主控台會建立代理程式設定檔，並且再次顯示包含新代理程式設定檔的連結的代理程式頁面。

    若要進行額外的代理程式設定檔配置，請按一下此連結，以顯示 [編輯] 代理程式頁面。
