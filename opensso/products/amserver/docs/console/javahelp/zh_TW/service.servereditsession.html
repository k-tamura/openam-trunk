階段作業

階段作業屬性可讓您為額外的網站和伺服器實例配置階段作業資訊。

階段作業限制

下列屬性會設定伺服器階段作業限制：

最大階段作業數

預設值為 5000。請指定允許的同步運作階段作業的最大數目。如果同步運作的階段作業最大值超過此數目，登入就會傳回「最大階段作業數」錯誤。

使階段作業無效的最長時間

預設值為 10。指定當階段作業建立後且使用者未登入時，從階段作業表格中移除該無效階段作業之前的期限 (以分鐘為單位)。此值應始終大於「認證」模組特性檔中的逾時值。

階段作業清除延遲

預設值為 60。指定清除階段作業操作的延遲時間 (以分鐘為單位)。在階段作業逾時之後，階段作業將在這段延長的時間內繼續位於階段作業伺服器上。用戶端應用程式使用此特性來檢查階段作業是否經由 SSO API 而逾時。這段延長的時間快結束時，階段作業就會被銷毀。如果使用者登出或 OpenSSO 元件明確銷毀階段作業，則階段作業不會在這段延長的時間內持續不變。階段作業在這段延長的時間內會處於「無效」狀態。

統計

下列屬性會設定統計配置：

記錄間隔

預設值為 60。指定兩次統計記錄之間經過的時間 (以分鐘為單位)。最小值為 5 秒，以避免 CPU 飽和。OpenSSO 會將任何少於 5 秒的值視為 5 秒。

狀態

預設值為「檔案」。指定統計記錄的位置。可能的值包括：

  • 關閉 — 不會記錄任何統計資料。

  • 檔案 — 統計資料會寫入所指定目錄下的某個檔案中。

  • 主控台 — 統計資料會寫入 Web 伺服器記錄檔中。

目錄

值會在安裝期間設定。範例：OpenSSO-base /server-URI/stats。指定建立除錯檔的目錄。

啟用主機查找

預設值為 false。啟用或停用在階段作業登入期間執行主機查找。

通知

下列屬性會設定通知配置：

通知池大小

預設值為 10。透過指定執行緒總數定義池的大小。

通知執行緒池臨界值

預設值為 100。指定工作佇列的最大長度。收到通知作業時，會將其傳送至作業佇列進行處理。如果佇列達到最大長度，之後收到的請求將會遭拒，並且發出 ThreadPoolException，直到佇列有空閒為止。

驗證

下列屬性會設定驗證配置：

不區分大小寫的用戶端 DN 比較

預設值為 true。比較代理程式 DN。如果值為 false，則比較時會區分大小寫。
