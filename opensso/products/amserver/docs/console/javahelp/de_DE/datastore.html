Data Stores

Als Data Store wird eine Datenbank bezeichnet, in der Benutzerattribute und Benutzerkonfigurationsdaten gespeichert werden können.

Access Manager bietet ein Identitäts-Repository-Plug-In, das eine Verbindung zu einem LDAPv3-Identitäts-Repository-Framework herstellt. Dieses neue Modell ermöglicht Ihnen Einsicht in Access Manager-Benutzerinformationen, ohne dass Sie Änderungen an Ihrer vorhandenen Benutzerdatenbank vornehmen müssen. Das Access Manager-Framework fasst die Daten des Identitäts-Repository-Plug-Ins mit den Daten anderer Access Manager-Plug-Ins zusammen und bildet so eine virtuelle Identität jedes einzelnen Benutzers. Diese universelle Identität kann Access Manager bei der Authentifizierung und Autorisierung für mehrere Identitäts-Repositories gemeinsam verwenden. Die virtuelle Benutzeridentität wird vernichtet, sobald die Benutzersitzung endet.

Informationen zur Erstellung eines neuen Data Stores und zur Konfiguration des Repository-Plug-Ins finden Sie unter So erstellen Sie einen neuen LDAPv3-Data Store.

Wenn Sie Access Manager im Bereichsmodus installiert haben, können Sie Access Manager SDK zur Verwaltung der Directory Server-Daten konfigurieren. Siehe Erstellen eines neuen Access Manager-Repository-Plug-Ins.

So erstellen Sie einen neuen LDAPv3-Data Store

  1. Wählen Sie den Bereich aus, dem Sie einen neuen Data Store hinzufügen möchten.

  2. Klicken Sie auf die Registerkarte "Data Stores".

  3. Klicken Sie in der Liste der Data Stores auf "Neu".

  4. Geben Sie einen Namen für den Data Store ein.

  5. Definieren Sie die Attribute für das LDAPv3-Repository-Plug-In. Eine Beschreibung der Attribute finden Sie unter LDAPv3-Repository-Plug-In-Attribute.

  6. Klicken Sie auf "Fertig stellen"
