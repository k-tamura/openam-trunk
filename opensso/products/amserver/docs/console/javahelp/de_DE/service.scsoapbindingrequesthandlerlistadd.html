So konfigurieren Sie einen Anforderungs-Handler

Die RequestHandler-Schnittstelle muss serverseitig von jedem Liberty-basierten Identitätsdienst implementiert sein, der den SOAP Binding-Dienst nutzt. Das Attribut "Anforderungs-Handler-Liste" speichert Informationen über die Implementierungsklassen der Webdienste, die den Anforderungs-Handler implementieren. The Request Handler List displays entries that contain key/value pairs separated by a pipe as in: key=disco|class=com.example.identity.liberty.ws.disco.DiscoveryService .

  1. Klicken Sie auf "Neu", um die neuen Anforderungshandler-Attribute anzuzeigen, oder klicken Sie auf einen konfigurierten Schlüsselwert, um vorhandene Attribute zu bearbeiten.
  2. Legen Sie die Attributwerte entsprechend der folgenden Informationen fest:
    key

    Dieser erforderliche Parameter ist der letzte Teil des URI-Pfads eines SOAP-Endpunkts. In OpenSSO ist der SOAP-Endpunkt das SOAP-Empfänger-Servlet. Die URI des SOAP-Empfängers lautet protocol ://host:port/ deploy-uri/Liberty/key. If you define disco as the key, the URI path to the SOAP endpoint for the corresponding Discovery-Dienst would be: protocol://host:port /amserver/Liberty/disco. Die einzelnen Dienstclients verwenden beim Verbindungsaufbau mit dem SOAP-Empfänger unterschiedliche Schlüssel.

    class

    Dieser erforderliche Parameter gibt den Namen der Implementierungsklasse des Anforderungs-Handlers für den jeweiligen Identitätsdienst an. For example: class=com.example.identity.liberty.ws.disco.DiscoveryService .

    SOAP-Vorgang

    Dieser Parameter enthält einen optionalen SOAP-spezifischen URI.

  3. Klicken Sie auf "OK", um die Konfiguration des Anforderungs-Handlers abzuschließen.
  4. Klicken Sie auf der SOAP Binding-Dienst-Seite auf "Speichern", um die Konfiguration des Dienstes abzuschließen.
