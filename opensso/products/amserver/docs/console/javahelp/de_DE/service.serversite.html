Server und Sites

Mit dem Konfigurationsattribut "Servers und Sites" ist die zentralisierte Konfigurationsverwaltung der Sites und Server für die gesamte Bereitstellung möglich.

Mehrere (zwei oder mehr) OpenSSO-Instanzen können auf mindestens zwei verschiedenen Hostservern bereitgestellt werden. Sie könnten beispielsweise zwei Instanzen auf einem Server und eine dritte Instanz auf einem weiteren Server bereitstellen. Oder Sie könnten alle Instanzen auf verschiedenen Servern bereitstellen. Außerdem können Sie die OpenSSO-Instanzen im Sitzungs-Failover-Modus konfigurieren, falls dies für Ihre Bereitstellung erforderlich ist.

Ein oder mehrere Lastenausgleichssysteme leiten Clientanforderungen an die verschiedenen OpenSSO-Instanzen weiter. Sie konfigurieren die einzelnen Lastenausgleichssysteme gemäß Ihren Bereitstellungsanforderungen (z. B. zur Verwendung des Round-Robin-Verfahrens oder des Lastendurchschnitts) zur Verteilung der Last zwischen den OpenSSO-Instanzen. Ein Lastenausgleichssystem vereinfacht die Bereitstellung und löst Probleme wie eine Firewall zwischen Client und Back-End-OpenSSO-Servern. Mit der OpenSSO-Bereitstellung kann ein Hardware- oder Software-Lastenausgleichssystem verwendet werden. Alle OpenSSO-Instanzen greifen auf denselben Directory Server zu.
