So erstellen Sie eine neue Authentifizierungsdomäne

Führen Sie die folgenden Schritte aus, um eine neue Authentifizierungsdomäne zu erstellen. Der Ausgangspunkt ist der Eintrag für Authentifizierungsdomänen der Verbindungsschnittstelle.

  1. Klicken Sie auf "Neu", um die Attribute für Authentifizierungsdomänen anzuzeigen.

    Die Profilseite für die neue Authentifizierungsdomäne wird angezeigt.

  2. Geben Sie einen Namen für die Authentifizierungsdomäne ein.
  3. Geben Sie im Feld "Beschreibung" eine Beschreibung der Authentifizierungsdomäne ein.
  4. Geben Sie einen Wert für den URL des Schreibprogramm-Dienstes ein.

    Der URL des Schreibprogramm-Dienstes gibt den Pfad des Servlets an, das das allgemeine Domänencookie erstellt. Verwenden Sie das Format http://common-domain-host :port/common/writer.

  5. Geben Sie einen Wert für den URL des Leseprogramm-Dienstes ein.

    Der URL des Leseprogramm-Dienstes gibt den Pfad des Servlets an, das das allgemeine Domänencookie liest. Verwenden Sie das Format http://common-domain-host :port/common/transfer.

  6. Wählen Sie "Aktiv" oder "Inaktiv".

    Der Standardstatus ist "Aktiv". Durch Auswahl von "Inaktiv" deaktivieren Sie die Kommunikation innerhalb der Authentifizierungsdomäne.

  7. Klicken Sie auf "OK", um die Konfiguration abzuschließen.

    Die neue Authentifizierungsdomäne wird nun im Fenster für Authentifizierungsdomänen angezeigt.
