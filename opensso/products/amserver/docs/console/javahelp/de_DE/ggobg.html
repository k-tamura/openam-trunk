So erstellen Sie eine neue Gruppe

Führen Sie diese Aufgabe, sofern erforderlich, an der Federated Access Manager -Administrationskonsole aus. Diese Aufgabe bezieht sich auf Webagentengruppen und J2EE-Agentengruppen.

Erstellen Sie eine Gruppe, wenn die Agenten bestimmte Eigenschaften aus der Gruppe übernehmen sollen. J2EE-Agenten können Eigenschaften aus einer J2EE-Agentengruppe übernehmen, Webagenten aus einer Webagentengruppe.

  1. Klicken Sie auf der Registerkarte für die Konfiguration auf "Agenten".
  2. Wählen Sie gegebenenfalls die Registerkarte für den entsprechenden Agententyp aus.
    • Webagenten:

      Die Registerkarte "Web" bezieht sich auf Webagenten und ist standardmäßig ausgewählt. Es ist keine weitere Auswahl erforderlich.

    • J2EE-Agenten:

      Die Registerkarte "J2EE" bezieht sich auf J2EE-Agenten. Klicken Sie bei Bedarf auf diese Registerkarte.

  3. Klicken Sie im Abschnitt "Gruppe" auf "Neu".

    Geben Sie im Feld "Name" den Namen der neuen Gruppe ein.

  4. Geben Sie im Feld für die Server-URL die Server-URL des Federated Access Manager ein.
  5. Klicken Sie auf "Erstellen".

    Die Konsole erstellt die Agentengruppe und zeigt die Agentenseite (WebAgent bzw. J2EEAgent) mit einer Verknüpfung zu der Gruppe an.

    Wenn Sie weitere Konfigurationen an der Gruppe vornehmen möchten, klicken Sie auf diese Verknüpfung, um die Seite zur Bearbeitung der Agentengruppe anzuzeigen.

    Zur Konfiguration können Sie dieselben Attribute festlegen wie bei einzelnen Agenten, außer dass die Attribute "Gruppe", "Passwort" und "Passwortbestätigung" auf Gruppenebene nicht verfügbar sind.

    Hinweis – Denken Sie außerdem daran, dass einigen Gruppenattributen bereits Variablenwerte zugewiesen wurden, die in den meisten Fällen nicht geändert werden sollten. Im Folgenden finden Sie ein Beispiel für einen solchen Wert:

    @AGENT_PROTO@://@AGENT_HOST@:@AGENT_PORT@/amagent
