Anforderungs-Handler-Liste

Zur Verarbeitung einer Anforderungsnachricht und zur Rückgabe einer Antwort stellt der SOAP Binding-Dienst die RequestHandler-Schnittstelle bereit. Die Schnittstelle muss serverseitig von jedem Liberty-basierten Identitätsdienst implementiert sein, der den SOAP Binding-Dienst nutzt. Das Attribut "Anforderungs-Handler-Liste" speichert Informationen über die Implementierungsklassen der Webdienste, die den Anforderungs-Handler implementieren. The Request Handler List displays entries that contain key/value pairs separated by a pipe as in: key=disco|class=com.example.identity.liberty.ws.disco.DiscoveryService . Die Parameter werden in der folgenden Tabelle erläutert.

Parameter

Kopfzeile

key

Der erforderliche key-Parameter ist der letzte Teil des URI-Pfads eines SOAP-Endpunkts. In OpenSSO ist der SOAP-Endpunkt das SOAP-Empfänger-Servlet. The URI to the SOAP Receiver is: protocol://host:port /deploy-uri/Liberty/key If you define disco as the key, the URI path to the SOAP endpoint for the corresponding Discovery-Dienst would be: protocol://host:port /amserver/Liberty/disco. Die einzelnen Dienstclients verwenden beim Verbindungsaufbau mit dem SOAP-Empfänger unterschiedliche Schlüssel.

class

Der erforderliche class-Parameter gibt den Namen der Implementierungsklasse des Anforderungs-Handlers für den jeweiligen Identitätsdienst an. For example: class=com.example.identity.liberty.ws.disco. DiscoveryService .

Informationen zum Hinzufügen eines neuen Anforderungs-Handlers finden Sie unter So konfigurieren Sie einen Anforderungs-Handler.
