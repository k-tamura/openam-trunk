Para configurar un identificador de sitio

Este atributo define cualquier sitio alojado por el servidor en el que esté instalado OpenSSO. Durante la instalación se define para el host un valor predeterminado y un Id. de sitio generado de forma automática. Se pueden especificar varias entradas. Por ejemplo, deberían definirse tanto el equilibrado de carga como varias instancias de OpenSSO que compartan la misma instancia de Directory Server. El punto de partida es el atributo Identificadores de sitio de la pantalla SAML en la interfaz Federación. Los Id. de sitio se definen en la pantalla de configuración de Servidores y sitios. Para obtener más información, consulte Servidores y sitios.

  1. Haga clic en Nuevo para agregar un identificador de sitio nuevo o haga clic en el nombre de un identificador de sitio que ya esté configurado para modificar su perfil.

    Se muestran los atributos de identificadores de sitio.

  2. Proporcione los valores de los atributos del identificador de sitio según la siguiente información:
    Id. de instancia

    El valor de esta propiedad es protocol ://host: port.

    Si configura SAML para SSL (tanto en el sitio de origen como de destino), asegúrese de que el protocolo definido sea https//.

    Id. de sitio

    El Id. de sitio es un identificador generado para cada sitio (aunque el valor sea el mismo para varios servidores detrás de un equilibrador de carga). Hay una clase en el paquete com.sun.identity.saml.common que se puede utilizar para generar este identificador de forma manual, en caso de ser necesario. Escriba lo siguiente en la línea de comandos:

    % java -classpath FM-classpath com.sun.identity.saml.common.SAMLSiteID protocol://host:port
    Nombre de emisor

    El valor predeterminado de esta propiedad es host :port, aunque podría ser cualquier URI.

  3. Haga clic en Aceptar para completar la configuración del identificador de sitio.
  4. Haga clic en Guardar en la página Propiedades del sitio local para completar la configuración de SAML.
