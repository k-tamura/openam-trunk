Almacenes de datos

Un almacén de datos es una base de datos en la que se pueden almacenar atributos de usuarios y datos de configuración de usuario.

Access Manager proporciona un complemento de repositorio de identidades que se conecta a una estructura de repositorio de identidades LDAPv3. Este nuevo modelo permite ver y recuperar la información de usuario de Access Manager sin necesidad de realizar cambios en la base de datos de usuario existente. La estructura de Access Manager integra los datos del complemento de repositorio de identidades con los datos de otros complementos de Access Manager para formar una identidad virtual para cada usuario. Access Manager puede utilizar la identidad universal en los procesos de autenticación y autorización en varios repositorios de identidades. La identidad de usuarios virtual se destruye al finalizar la sesión del usuario.

Consulte Para crear un nuevo almacén de datos LDAPv3 para obtener información sobre cómo crear un nuevo almacén de datos y configurar el complemento de repositorio.

Si ha instalado Access Manager en el modo de dominios, puede configurar el SDK de Access Manager para administrar los datos de Directory Server. Consulte Creación de un nuevo complemento de repositorio de Access Manager.

Para crear un nuevo almacén de datos LDAPv3

  1. Seleccione el dominio para el que desea agregar un nuevo almacén de datos.

  2. Haga clic en la ficha Data Store (Almacén de datos).

  3. Haga clic en New (Nuevo) en la lista de almacenes de datos.

  4. Introduzca un nombre para el almacén de datos.

  5. Defina los atributos para el complemento de repositorio LDAPv3. Para obtener una descripción de los atributos, consulte Atributos del complemento de repositorio LDAPv3.

  6. Haga clic en Finish (Finalizar).
