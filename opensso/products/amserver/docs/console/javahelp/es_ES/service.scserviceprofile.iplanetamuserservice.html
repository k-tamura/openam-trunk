Usuario

Las preferencias de usuario predeterminadas se definen mediante el servicio de usuario. Éstas incluyen la zona horaria, el idioma y la vista de inicio de DN. Los atributos del servicio de usuario son atributos dinámicos.

  • Idioma preferido por el usuario

  • Zona horaria preferida por el usuario

  • Idioma heredado

  • Vista de inicio de administrador

  • Estado de usuario predeterminado

Idioma preferido por el usuario

Este campo especifica el idioma del texto seleccionado por el usuario, que se muestra en la consola de OpenSSO. El valor predeterminado es en. Este valor asigna un conjunto de claves de localización a la sesión de usuario de forma que el texto en pantalla aparece en el idioma adecuado para el usuario.

Zona horaria preferida por el usuario

Este campo especifica la zona horaria en la que el usuario accede a la consola de OpenSSO. No hay ningún valor predeterminado.

Idioma heredado

Este campo especifica la configuración regional del usuario. El valor predeterminado es en_US. Consulte Idiomas admitidos pata obtener una lista de las configuraciones regionales.

Vista de inicio de administrador

Si el usuario es un administrador de OpenSSO, este campo especifica el nodo que será el punto de inicio y se mostrará en la consola de OpenSSO cuando el usuario inicie una sesión. No hay ningún valor predeterminado. Se puede utilizar un DN válido para el que el usuario tenga, al menos, acceso de lectura.

Estado de usuario predeterminado

Esta opción indica el estado predeterminado para un usuario que se acaba de crear. El estado de entrada de usuario anula este estado. Sólo los usuarios activos se pueden autenticar mediante OpenSSO. El valor predeterminado es Activo. Se puede seleccionar cualquiera de estas opciones en el menú desplegable:

Activo

El usuario se puede autenticar mediante OpenSSO.

Inactivo

El usuario no puede autenticarse mediante OpenSSO, pero el perfil de usuario permanece almacenado en el directorio.

Para definir el estado de un usuario individual, debe registrarse el servicio de usuario, seleccionar el valor, asignarle un rol y agregar este rol al perfil del usuario.
