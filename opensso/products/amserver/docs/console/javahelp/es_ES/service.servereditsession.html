Sesión

Los atributos de sesión permiten configurar la información de sesión para instancias del servidor o sitios adicionales.

Límites de sesiones

Los siguientes atributos establecen los límites de sesiones del servidor:

Nº máximo de sesiones

El valor predeterminado es 5000. Especifica el número máximo de sesiones simultáneas permitidas. El inicio de sesión envía un error si el valor de número máximo de sesiones simultáneas supera esta cifra.

Tiempo máx. de sesión no válida

El valor predeterminado es 10. Especifica el número de minutos tras los que se eliminará la sesión no válida de la tabla de sesiones si se ha creado y el usuario no ha iniciado una sesión. Este valor debe ser siempre superior al valor de tiempo de espera especificado en el archivo de propiedades del módulo de autenticación.

Retraso de purga de sesión

El valor predeterminado es 60. Esta propiedad especifica el número de minutos de retraso de la operación de purga de sesión. Se trata de un periodo de tiempo ampliado, una vez agotado el tiempo de espera de la sesión, durante el que la sesión permanece en el servidor de sesiones. La aplicación de cliente utiliza esta propiedad para comprobar si se ha agotado el tiempo de espera de la sesión mediante las API de SSO. Al final del periodo de tiempo ampliado, se destruirá la sesión. La sesión no se conserva durante el periodo de tiempo ampliado si el usuario cierra la sesión o un componente de OpenSSO la destruye de forma explícita. La sesión presenta el estado INVALID (NO VÁLIDO) durante este periodo ampliado.

Estadísticas

Los siguientes atributos establecen la configuración estadística:

Intervalo de registro

El valor predeterminado es 60. Especifica el número de minutos transcurridos entre cada actividad de registro de estadísticas. El valor mínimo es de 5 segundos para evitar la saturación de la CPU. OpenSSO asume como 5 segundos cualquier valor inferior a éste.

Estado

El valor predeterminado es "file". Especifica la ubicación del registro de estadísticas. Los valores posibles son:

  • off (desactivado): no se registra ninguna estadística.

  • file (archivo): las estadísticas se escriben en el archivo ubicado en el directorio especificado.

  • console (consola): las estadísticas se escriben en el archivo de registro del servidor web.

Directorio

Este valor se establece durante la instalación. Ejemplo: OpenSSO-base /server-URI/stats. Especifica el directorio en el que se crean los archivos de depuración.

Habilitar búsqueda de host

El valor predeterminado es false. Habilita o deshabilita la búsqueda de host durante el registro de sesión.

Notificación

Los siguientes atributos establecen la configuración de notificación:

Tamaño del conjunto de notificación

El valor predeterminado es 10. Define el tamaño del conjunto especificando el número total de subprocesos.

Umbral del conjunto de subprocesos de notificación

El valor predeterminado es 100. Especifica la longitud máxima de la cola de tareas. Cada vez que se recibe una tarea de notificación, se envía a la cola de tareas para su procesamiento. Si la cola alcanza la longitud máxima, las solicitudes entrantes adicionales se rechazarán y se generará una excepción ThreadPoolException hasta que la cola tenga espacio libre.

Validación

Los siguientes atributos establecen la configuración de la validación:

Comparación de DN sin distinción entre mayúsculas y minúsculas

El valor predeterminado es "true" (verdadero). Compara el DN de agente. Si se establece en "false" (falso), la comparación distinguirá entre mayúsculas y minúsculas.
