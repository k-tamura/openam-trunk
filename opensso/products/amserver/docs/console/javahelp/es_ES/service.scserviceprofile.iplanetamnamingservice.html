Asignación de nombres

El servicio de asignación de nombres se utiliza para obtener y definir las direcciones URL, complementos y configuraciones, así como las notificaciones de solicitudes para diversos servicios de OpenSSO como sesión, autenticación, registro SAML y federación.

Este servicio permite que los clientes encuentren la dirección URL correcta del servicio si la plataforma está ejecutando más de una instancia de OpenSSO. Cuando se encuentra una dirección URL de asignación de nombres, este servicio descodificará la sesión del usuario y sustituirá dinámicamente el protocolo, el host y el puerto por los parámetros de la sesión. De esta forma, se garantiza que la dirección URL que el servicio devuelve está destinada al host en el que se creó la sesión del usuario. Los atributos de asignación de nombres son:

  • URL de servicio de perfil

  • URL de servicio de sesión

  • URL de servicio de registro

  • URL de servicio de política

  • Dirección URL del servicio de autenticación

  • Perfil Web SAML/URL de servicio Artefacto

  • URL de servicio SAML SOAP

  • Perfil Web SAML/URL de servicio POST

  • URL del servicio del administrador de aserción SAM

  • URL del servicio del administrador de aserción de federación

  • Dirección URL del administrador del token de seguridad

  • URL de punto final de JAXRPC

URL de servicio de perfil

Este campo contiene un valor igual a:

%protocol://%host:%port/Server_DEPLOY_URI/profileservice

Esta sintaxis permite la sustitución dinámica de la dirección URL del perfil en función de los parámetros de sesión específicos.
