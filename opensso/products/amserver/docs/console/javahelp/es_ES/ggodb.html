Atributos de enlace SOAP

Autenticador de servicio Web

Este atributo toma como valor la clase de implementación de la interfaz del autenticador de servicio Web. Esta clase autentica una solicitud y genera una credencial para un consumidor de servicios Web.

Mecanismos de autenticación admitidos

Este atributo especifica los mecanismos de autenticación admitidos por el receptor de SOAP. Los mecanismos de autenticación ofrecen autenticación de usuario, así como integridad de datos y cifrado. De forma predeterminada, todas los mecanismos de autenticación disponibles están seleccionados. Si no se selecciona ninguno y un consumidor de servicios Web (WXC) envía una solicitud mediante este método, se rechazará la solicitud. A continuación se muestra una lista de los mecanismos de autenticación admitidos:

  • urn:liberty:security:2003-08:null:null

  • urn:liberty:security:2003-08:null:X509

  • urn:liberty:security:2003-08:null:SAML

  • urn:liberty:security:2004-04:null:Bearer

  • urn:liberty:security:2003-08:TLS:null

  • urn:liberty:security:2003-08:TLS:X509

  • urn:liberty:security:2003-08:TLS:SAML

  • urn:liberty:security:2004-04:TLS:Bearer

  • urn:liberty:security:2003-08:ClientTLS:null

  • urn:liberty:security:2003-08:ClientTLS:X509

  • urn:liberty:security:2003-08:ClientTLS:SAML

  • urn:liberty:security:2004-04:ClientTLS:Bearer

Aplicar sólo proveedores conocidos

Si se habilita, la propiedad aplicará el encabezado ProviderID enviado en un mensaje SOAP para garantizar que exista el proveedor en el sistema. De lo contrario, se rechazará la solicitud. Si no se habilita este atributo, se omitirá la comprobación.

Alias de certificado para la autenticación de cliente SSL

Este valor se establece durante la instalación. El alias de certificado de cliente que se utilizará en la conexión SSL para el enlace SOAP de Liberty.

Límite de tiempo del mensaje obsoleto

El valor predeterminado es 300000. Determina si el mensaje está obsoleto y, por lo tanto, ya no es de confianza. Si la marca del tiempo del mensaje es anterior a la marca del tiempo actual en función del número de milisegundos especificado, el mensaje se considerará no válido.

Intervalo de vaciado de la caché de Id. de mensaje

El valor predeterminado es 60000. Especifica el número de milisegundos que deben transcurrir antes de que comiencen los eventos de limpieza de la caché. Cada mensaje se almacena en la caché con su propio Id. de mensaje para evitar que existan mensajes duplicados. Cuando el tiempo actual del mensaje menos el tiempo recibido supera el valor de límite de tiempo de caducidad, el mensaje se eliminará de la caché.

Actores SOAP admitidos

El valor predeterminado es http://schemas.xmlsoap.org/soap/actor/next . Especifica los actores SOAP admitidos. Cada actor debe separarse con un carácter de barra vertical (|).

Asignación de prefijo de espacio de nombre

El valor predeterminado es:

=S=http://schemas.xmlsoap.org/soap/envelope/|sb=urn:liberty:sb:2003-08
|pp=urn:liberty:id-sis-pp:2003-08|ispp=http://www.sun.com/identity/
liberty/pp|is=urn:liberty:is:2003-08

Especifica la asignación de prefijo de espacio de nombre cuando se realiza una operación "marshall" en un árbol de contenido JAXB para un árbol DOM. La sintaxis es prefix=namespace|prefix=namespace|...

Lista de paquetes JAXB

Especifica la lista de paquetes JAXB utilizados al crear JAXBContext . Cada paquete debe separarse mediante dos puntos (:).

Versión del servicio web de identidades Liberty

Esta propiedad determina el marco ID-WSF de Liberty cuando éste no puede determinarse desde el mensaje entrante o la oferta de recursos si OpenSSO está actuando como WSC. Los valores pueden ser 1.0 ó 1.1. El valor predeterminado es 1.1.
