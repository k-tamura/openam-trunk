UNIX

Este módulo exclusivo de Solaris permite realizar la autenticación mediante la identificación y la contraseña de UNIX de un usuario. Si se modifica alguno de los atributos de autenticación de UNIX, se deberá reiniciar tanto OpenSSO como el asistente amunixd. Los atributos de autenticación de UNIX son atributos globales y de dominio. Los atributos son:

  • Puerto de configuración

  • Puerto de autenticación

  • Tiempo de espera

  • Subprocesos

  • Nivel de autenticación

  • Nombre de servicio PAM

Puerto de configuración

Este atributo especifica el puerto en el que el auxiliar de UNIX recibe, durante el inicio, la información de configuración incluida en los atributos de puerto de autenticación del auxiliar de UNIX, tiempo de espera del auxiliar de UNIX y subprocesos del auxiliar de UNIX. El valor predeterminado es 58946.

Puerto de autenticación

Este atributo especifica el puerto en el que el auxiliar de UNIX recibe las solicitudes de autenticación tras la configuración. El valor predeterminado es 57946.

Tiempo de espera

Este atributo especifica el número de minutos de los que disponen los usuarios para completar la autenticación. Si los usuarios sobrepasan el tiempo asignado, la autenticación presenta un error automáticamente. El tiempo predeterminado es de 3 minutos.

Subprocesos

Este atributo especifica el número máximo permitido de sesiones de autenticación de UNIX simultáneas. Si se alcanza el número máximo en un momento dado, los siguientes intentos de autenticación no se permitirán hasta que se libere una sesión. El valor predeterminado es 5.

Nivel de autenticación

El nivel de autenticación se define de forma independiente para cada método de autenticación. El valor indica en qué medida se debe confiar en un mecanismo de autenticación. Una vez que el usuario se ha autenticado, este valor se guarda en el token SSO de la sesión. Cuando se presenta el token SSO a una aplicación a la que desea acceder el usuario, la aplicación utiliza el valor almacenado para determinar si el nivel es suficiente para permitir el acceso al usuario. Si el nivel de autenticación guardado en un token SSO no satisface el valor mínimo necesario, la aplicación puede solicitar al usuario que se autentique de nuevo mediante un servicio con un mayor nivel de autenticación. El valor predeterminado es 0.

Nota – Si no se especifica ningún nivel de autenticación, el token SSO almacena el valor especificado en el Nivel de autenticación predeterminado del atributo de autenticación "Principal".

Nombre de servicio PAM

Define la configuración o la pila del Módulo de autenticación conectable (PAM, siglas en inglés) incluida en el sistema operativo y utilizada para la autenticación de UNIX. En Solaris, el nombre es normalmente other y, en Linux, es password.
