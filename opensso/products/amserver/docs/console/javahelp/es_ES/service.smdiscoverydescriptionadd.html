Para configurar una descripción de servicio

El atributo de descripción de servicio define un servicio Web que se está ejecutando en un punto final de protocolo distinto. Se define al configurar Ofertas de recursos para bootstrapping. La información sobre las instancias de servicio debe comunicarse en distintos contextos. Por ejemplo, el Servicio de detección definido es un servicio de identidades que proporciona una enumeración de las ofertas de recursos (cada una de las cuales incluye una descripción de instancia de servicio).

  1. Haga clic en Add Description (Agregar descripción) para ver los nuevos atributos de descripción de servicio o haga clic en Editar para modificar una descripción existente.
  2. Proporcione los valores de los atributos según la siguiente información:
    Id. de mecanismo de seguridad

    Este atributo permite agregar los métodos de autenticación admitidos por el Servicio de detección. Estos mecanismos de seguridad hacen referencia a la forma en la que el consumidor de servicios Web se autentica al proveedor de servicios Web o proporciona seguridad en los mensajes. De forma predeterminada están seleccionados todos los métodos disponibles que admite la instancia de servicios. Si no se selecciona un método de autenticación y un consumidor de servicios Web envía una solicitud mediante este método, la solicitud será rechazada. Consulte Mecanismos de autenticación admitidos.

    URL de punto final

    El valor de este atributo es la URL de punto final de SOAP a través de HTTP. El esquema del URI debe ser HTTP o HTTPS, como en https://soap.profile-provider.com/soap.

    Acción SOAP

    El valor de este atributo opcional es equivalente al atributo wsdlsoap:soapAction del elemento wsdlsoap:operation de la descripción específica basada en WSDL del servicio.

  3. Haga clic en Aceptar para completar la configuración del servicio.
  4. Haga clic en Guardar en la página del Servicio de detección para completar la configuración del servicio.
