Para crear un nuevo grupo

Si lo desea, realice esta tarea en la consola de administración de Federated Access Manager . Esta tarea está destinada a los grupos de agentes web y J2EE.

Cree un grupo si desea que los agentes hereden propiedades específicas del grupo. Los agentes J2EE pueden heredar propiedades de un grupo de agentes J2EE, mientras que los agentes web pueden heredar propiedades de un grupo de agentes web.

  1. Haga clic en Agentes en la ficha Configuración.
  2. Si es necesario, seleccione la ficha del tipo de agente adecuado.
    • Agentes web:

      La ficha Web se utiliza para los agentes web y se selecciona de forma predeterminada. No es necesario realizar ninguna selección adicional.

    • Agentes J2EE:

      La ficha J2EE se utiliza para los agentes J2EE. Si es pertinente, haga clic en esta ficha.

  3. En la sección Grupo, haga clic en Nuevo.

    En el campo Nombre, introduzca el nombre del nuevo grupo.

  4. En el campo URL del servidor, introduzca la URL del servidor de Federated Access Manager.
  5. Haga clic en Create (Crear).

    La consola crea el grupo de agentes y muestra de nuevo la página del agente (agente web o J2EE) con un vínculo al grupo.

    Si desea realizar la configuración adicional del grupo, haga clic en este vínculo para que se muestre la página Editar grupo de agentes.

    Los atributos que puede establecer para configurar un grupo son iguales a los que se utilizan para un agente individual, a excepción de los atributos Grupo, Contraseña y Confirmar contraseña, que no están disponibles en el nivel de grupo.

    Nota – Además, tenga en cuenta que algunos atributos de grupos ya tienen valores de variables asignados que, en la mayoría de los casos, no se pueden modificar. A continuación, se muestra un ejemplo de un valor de este tipo:

    @AGENT_PROTO@://@AGENT_HOST@:@AGENT_PORT@/amagent
