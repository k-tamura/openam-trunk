Para agregar miembros a una entidad de afiliación

Después de crear una entidad de afiliación, puede agregar miembros a la afiliación. El punto de partida es Entidades en la interfaz Federación.

Nota – También puede modificar los atributos comunes en esta pantalla.

  1. Seleccione el nombre de una entidad de afiliación configurada para modificar su perfil.

    Aparece la página de perfil de la entidad.

  2. Elija Affiliate (Afiliado) en el menú View (Ver).
  3. Seleccione cualquiera de las entidades disponibles para asignarla a la afiliación.
  4. Modifique los valores de los atributos comunes.

    Los atributos comunes contienen valores que definen generalmente el propio proveedor de identidades.

    Nombre

    Escriba un nuevo URI para la afiliación si es necesario. El valor actual de este atributo es el URI asignado al atributo Nombre de afiliado al crear la entidad. Cualquier valor nuevo debe ser exclusivo entre los nombres de afiliados.

    Propietario

    Escriba un nuevo URI para definir un operador principal o propietario diferente para la afiliación. El valor actual de este atributo es el URI asignado al atributo Propietario de afiliado al crear la entidad. El nuevo valor de este atributo es un URI que coincidirá con el valor definido en el atributo Nombre de entidad de la entidad a la que pertenece el nuevo proveedor/propietario. Cualquier valor nuevo debe ser exclusivo entre los nombres de afiliados.

    Nota – Si se define un proveedor como propietario de afiliado, esto no implica que se incluya automáticamente como miembro de la afiliación. Si un propietario es también miembro, el URI debe definirse en esta ubicación como propietario y, dentro de la propia entidad de afiliación, como miembro.

    Clave de firma: alias de clave

    Los certificados se guardan en un archivo de almacén de claves de Java. Cada certificado del archivo se asigna a un alias para recuperarlo. Escriba el alias de clave asociado al certificado que se utiliza para firmar las solicitudes y las respuestas.

    Clave de cifrado: alias de clave

    Los certificados se guardan en un archivo de almacén de claves de Java. Cada certificado del archivo se asigna a un alias para recuperarlo. Escriba el alias de clave asociado al certificado de seguridad del proveedor.

    Clave de cifrado: tamaño de clave

    Escriba la longitud de las claves que utiliza el consumidor del servicio Web al interactuar con otra entidad.

    Clave de cifrado: método de cifrado

    Escriba un URI que señale a la preferencia de método de cifrado

  5. Haga clic en Guardar.
