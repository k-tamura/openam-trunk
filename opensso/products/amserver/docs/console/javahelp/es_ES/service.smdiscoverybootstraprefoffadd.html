Para configurar una oferta de recursos para Bootstrapping

Sólo está permitida una oferta de recursos para bootstrapping; de forma predeterminada, esta oferta contiene información relacionada con el Servicio de detección. Si ya se ha definido una oferta de recursos, puede modificar los atributos haciendo clic en su nombre. También puede seleccionar la casilla situada al lado del nombre y eliminar la oferta de recursos existente. Para configurar una nueva oferta de recursos, debería hacer clic en Nuevo.

  1. Haga clic en “Nuevo” (New) para mostrar los nuevos atributos del asignador o seleccione el nombre de un asignador configurado para modificarlo.
  2. Escriba una descripción opcional para la oferta de recursos.
  3. Proporcione los valores de los atributos de la instancia del servicio según la siguiente información:
    Tipo de servicio

    Este atributo toma como valor un URI que define el tipo de servicio que implementa la oferta de recursos. Por ejemplo, urn:liberty:disco:2003-08.

    Nota – Se recomienda que este URI sea el mismo que el URI de targetNamespace de la descripción WSDL abstracta para el servicio.

    Provider ID

    Este atributo toma como valor un URI del proveedor de la instancia de servicios. Por ejemplo, http://daiquiri.sun.com:80/amserver/Liberty/disco .

    Descripción del servicio

    Estos atributos definen un servicio Web que se está ejecutando en un punto final de protocolo distinto. La información sobre las instancias de servicio debe comunicarse en distintos contextos. Por ejemplo, el Servicio de detección definido en esta especificación es un servicio de identidades que proporciona una enumeración de las ofertas de recursos (cada una de las cuales incluye una descripción de la instancia de servicio). Consulte Para configurar una descripción de servicio.

  4. Proporcione los valores de los atributos de opciones de ofertas de recursos según la siguiente información:
    Opciones

    Marque esta casilla si el servicio no tiene opciones disponibles para la oferta de recursos. Las opciones proporcionan indicaciones a un solicitante potencial sobre si determinados datos u operaciones pueden estar disponibles con una oferta concreta de recursos. Por ejemplo, se puede proporcionar una opción que indique que hay disponible información de contacto personal.

    Lista de opciones

    Este atributo contiene una lista de las opciones para la instancia del servicio. La opción se define como un URI. El conjunto de posibles URI está normalmente estandarizado por el tipo de servicio.

  5. Habilita las directivas en función de la siguiente información:
    Referencias de Id. de descripción para: GenerateBearerToken

    Todas las Directivas admitidas pueden incluir una referencia descriptiva. Si el atributo de referencias de ID no está habilitado para una directiva, ésta se aplicará a todos los mecanismos de autenticación proporcionados en la oferta de recursos. Si el atributo de referencias de ID está habilitado para una directiva, DEBE incluir una lista de referencias de ID que señale el mecanismo de autenticación al que está asociada la directiva. Si el atributo está presente, la directiva sólo DEBE aplicarse a aquellas descripciones a las que se hace referencia en la lista de referencias de ID. Esto puede ser útil si algunas directivas son incompatibles con algunos mecanismos de seguridad.
