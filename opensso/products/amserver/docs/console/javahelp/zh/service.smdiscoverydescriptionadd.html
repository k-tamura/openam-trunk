配置服务说明

“服务说明”属性用于在不同的协议端点定义正在运行的 Web 服务。 它在配置用于引导的资源提供时进行定义。有关服务实例的信息需要在各种语境中进行传送。例如，定义的搜索服务是身份服务，它用于枚举资源提供（每个资源提供都包含一个服务实例说明）。

  1. 单击“添加说明”以显示新的服务说明属性，或单击“编辑”修改现有说明。
  2. 基于以下信息，为属性提供值：
    安全机制 ID

    搜索服务支持的验证方法可添加到该属性中。这些安全机制指的是 Web 服务使用方向 Web 服务提供者进行验证或提供消息级安全的方法。默认情况下，服务实例支持的所有可用方法都处于选定状态。如果 Web 服务使用方使用未选定的验证方法发送请求，则系统将拒绝该请求。参见支持的验证机制。

    端点 URL

    该属性的值为 SOAP-over-HTTP 端点的 URL。URI 方案必须为类似 https://soap.profile-provider.com/soap 中的 HTTP 或 HTTPS。

    SOAP 操作

    该可选属性的值与服务基于 WSDL 的具体说明中 wsdlsoap:operation 元素的 wsdlsoap:soapAction 属性相同。

  3. 单击“确定”完成服务的配置操作。
  4. 单击“搜索服务”页面上的“保存”完成服务的配置操作。
