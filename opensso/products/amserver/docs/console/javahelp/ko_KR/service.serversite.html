서버 및 사이트

서버 및 사이트 구성 속성을 사용하면 전체 배포를 위한 사이트 및 서버의 중앙 집중화된 구성 관리가 가능합니다.

둘 이상의 여러 OpenSSO 인스턴스를 둘 이상의 서로 다른 호스트 서버에 배포할 수 있습니다. 예를 들어 한 서버에 두 개의 인스턴스를 배포하고 세 번째 인스턴스는 다른 서버에 배포하거나 모든 인스턴스를 서로 다른 서버에 배포할 수 있습니다. 또한 배포에 필요한 경우 세션 페일오버 모드에서 OpenSSO 인스턴스를 구성할 수 있습니다.

하나 이상의 로드 밸런서가 클라이언트 요청을 여러 OpenSSO 인스턴스로 라우팅합니다. 배포 요구 사항(예: 라운드 로빈 또는 평균 로드)에 따라 각 로드 밸런서를 구성하여 OpenSSO 인스턴스 사이의 로드를 분산합니다. 로드 밸런서는 배포를 간소화하며 클라이언트와 백엔드 OpenSSO 서버 간의 방화벽과 같은 문제를 해결합니다. OpenSSO 배포에 하드웨어 또는 소프트웨어 로드 밸런서를 사용할 수 있습니다. 모든 OpenSSO 인스턴스는 같은 Directory Server에 액세스합니다.
