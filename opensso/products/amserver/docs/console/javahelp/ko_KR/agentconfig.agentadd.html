새 에이전트를 만들려면

새 에이전트를 만드는 것은 새 에이전트 프로필을 만드는 것입니다. OpenSSO 관리 콘솔에서 이 작업을 수행할 수 있습니다. 개별 단계 중 일부는 J2EE 및 웹 정책 에이전트 유형에만 적용됩니다.

  1. [액세스 제어] 탭을 누릅니다.
  2. 에이전트가 속할 영역 이름을 누릅니다.
  3. [에이전트] 탭을 누릅니다.
  4. 적절한 에이전트 유형에 해당하는 탭을 선택합니다.
    • 웹 에이전트
    • J2EE 에이전트
    • 웹 서비스 공급자 에이전트
    • 웹 서비스 클라이언트 에이전트
    • STS 클라이언트 에이전트

      STS(Security Token Service) 클라이언트 에이전트 유형은 검색 에이전트와 STS 에이전트 중에서 선택할 수 있습니다. 해당 에이전트 유형에 대한 자세한 내용은 STS 클라이언트를 참조하십시오.

    • 2.2 에이전트:

      [2.2 에이전트] 탭은 정책 에이전트 2.2 소프트웨어 집합에서 에이전트용으로 사용됩니다.

    • 에이전트 인증자
  5. [에이전트] 섹션에서 [새로 만들기]를 누릅니다.
  6. [이름] 필드에 새 에이전트 프로필의 이름을 입력합니다.
  7. 비밀번호를 입력하고 확인합니다.

    주의 – 이 비밀번호는 agentadmin 프로그램을 실행하여 에이전트를 설치할 때 지정한 에이전트 프로필 비밀번호 파일에 입력한 비밀번호와 동일해야 합니다.

    8–10단계는 웹 및 J2EE 정책 에이전트에만 적용됩니다.

  8. 적용 가능한 구성을 로컬 또는 중앙 집중화됨 중에서 선택합니다 .

    로컬 구성을 선택한 경우 이 에이전트 관련 등록 정보를 콘솔에서 편집할 수 있습니다. 이 시나리오에서 에이전트는 에이전트 설치 디렉토리에 있는 로컬 파일인 OpenSSOAgentBootstrap.properties 및 OpenSSOAgentConfiguration.properites에서 구성 정보를 검색합니다. 로컬로 구성된 에이전트의 경우 OpenSSOAgentConfiguration.properites 파일을 직접 편집하여 등록 정보 값을 변경하십시오.

  9. [서버 URL] 필드에 OpenSSO 서버 URL을 입력합니다.

    예를 들면 다음과 같습니다.

    http://OpenSSO_EnterpriseHost.example.com:8080/OpenSSO
  10. [에이전트 URL] 필드에 에이전트 응용 프로그램(agentapp)의 URL을 입력합니다.

    예를 들면 다음과 같습니다.

    웹 에이전트

    http://agentHost.example.com:8090

    J2EE 에이전트

    http://agentHost.example.com:8090/agentapp

  11. [만들기]를 누릅니다.

    콘솔에서 에이전트 프로필을 만들고 새로운 에이전트 프로필에 대한 링크와 함께 에이전트 페이지를 다시 표시합니다.

    에이전트 프로필을 추가로 구성하려면 이 링크를 눌러 에이전트 편집 페이지를 표시합니다.
