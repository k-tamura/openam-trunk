새 그룹을 만들려면

필요한 경우 Federated Access Manager 관리 콘솔에서 이 작업을 수행합니다. 이 작업은 웹 에이전트 그룹 및 J2EE 에이전트 그룹에 적용됩니다.

에이전트가 그룹에서 특정 등록 정보를 상속받도록 하려면 그룹을 만듭니다. J2EE 에이전트는 J2EE 에이전트 그룹에서 등록 정보를 상속받을 수 있으며, 웹 에이전트는 웹 에이전트 그룹에서 등록 정보를 상속받을 수 있습니다.

  1. [구성] 탭의 [에이전트]를 누릅니다.
  2. 필요한 경우 적절한 에이전트 유형에 해당하는 탭을 선택합니다.
    • 웹 에이전트:

      [웹] 탭은 웹 에이전트용이며, 기본적으로 선택됩니다. 다른 선택은 필요하지 않습니다.

    • J2EE 에이전트:

      [J2EE] 탭은 J2EE 에이전트용입니다. 해당하는 경우 이 탭을 누릅니다.

  3. 그룹 선택에서 [새로 만들기]를 누릅니다.

    [이름] 필드에 새 그룹의 이름을 입력합니다.

  4. [서버 URL] 필드에 Federated Access Manager 서버 URL을 입력합니다.
  5. [만들기]를 누릅니다.

    콘솔이 에이전트 그룹을 만들고 해당 그룹에 대한 링크가 있는 에이전트(WebAgent 또는 J2EEAgent) 페이지가 다시 표시됩니다.

    그룹을 추가로 구성하려면 이 링크를 눌러 에이전트 그룹 편집 페이지를 표시합니다.

    그룹을 구성하기 위해 설정할 수 있는 속성은 그룹 수준에서 그룹, 비밀번호 및 비밀번호 확인 속성을 사용할 수 없는 경우를 제외하고 개별 에이전트와 동일합니다.

    주 – 또한 일부 그룹 속성에는 대부분의 경우 변경하지 않아야 하는 변수 값이 이미 할당되어 있습니다. 다음은 이러한 값의 한 예입니다.

    @AGENT_PROTO@://@AGENT_HOST@:@AGENT_PORT@/amagent
