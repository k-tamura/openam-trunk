SAMLv2 SOAP 바인딩

SAMLv2 SOAP 바인딩 서비스는 OpenSSO 클라이언트와 OpenSSO 서버 간에 SAMLv2 요청 및 응답 메시지의 SOAP 기반 교환을 제공합니다. 수신된 요청은 요청 처리기로 위임되어 추가적인 처리를 진행합니다. Request Handler와 메타 별칭의 키는 SOAP 바인딩 서비스 URL에 있습니다. 메타 별칭과 RequestHandler의 매핑은 SAMLv2 SOAP 바인딩 서비스에 저장되며, OpenSSO 구성 저장소에서 읽을 수 있습니다.

요청 처리기 목록

RequestHandlerList는 RequestHandler 구현에 대한 메타 별칭 매핑이 포함된 키/값 항목 목록입니다. OpenSSO 8.0 서버가 PDP(Policy Decision Point) 역할을 하도록 구성하는 경우 이 속성을 설정해야 합니다.

키는 정책 결정 지점(PDP) 메타 별칭입니다. 클래스는 Java 클래스 이름으로, XACML 요청을 처리할 수 있는 RequestHandler 인터페이스의 구현입니다.

예를 들어 XACML 정책 결정 지점(PDP)의 메타 별칭이 /pdp이고 해당 인터페이스의 구현이 com.sun.identity.xacml.plugins.XACMLAuthzDecisionQueryHandler인 경우 키는 /pdp로 설정되고 클래스는 com.sun.identity.xacml.plugins.XACMLAuthzDecisionQueryHandler로 설정되어야 합니다.
