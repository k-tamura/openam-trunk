Attribution de nom

Le service Attribution de nom permet d'obtenir et de définir des URL, des plug-ins et des configurations, ainsi que des notifications de demande pour divers autres services d'OpenSSO (session, authentification, journalisation, SAML et Federation).

Ce service permet aux clients de trouver l'URL de service correct si la plate-forme exécute plusieurs instances d'OpenSSO. Lorsqu'un URL d'attribution de nom est trouvé, le service Attribution de nom décode la session de l'utilisateur et remplace de manière dynamique le protocole, l'hôte, le port par les paramètres de la session. Ceci garantit que l'URL retourné par la service correspond à l'hôte sur lequel est créée la session utilisateur. Les attributs Attribution de nom sont les suivants :

  • URL du service de profil

  • URL du service de session

  • URL du service de journalisation

  • URL du service de modalité

  • URL du service d'authentification

  • URL du service profil Web SAML/artéfact SAML

  • URL du service SOAP SAML

  • URL du service profil Web SAML/POST SAML

  • URL du service du gestionnaire d'assertions SAML

  • URL du service du gestionnaire d'assertions de fédérations

  • URL du gestionnaire de jeton de sécurité

  • URL d'arrivée JAXRPC

URL du service de profil

Ce champ accepte une valeur du type :

%protocole://%hôte:%port/URI_déploiement_serveur/profileservice

Cette syntaxe permet le remplacement dynamique de l'URL de profil sur la base des paramètres de session spécifiques.
