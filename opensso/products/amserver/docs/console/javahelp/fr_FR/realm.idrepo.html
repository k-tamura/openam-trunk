Magasins de données

Un magasin de données est une base de données dans laquelle vous pouvez stocker des attributs utilisateur et des données de configuration utilisateur.

OpenSSO fournit des plug-ins du référentiel d'identités qui se connectent à une structure de référentiel d'identités LDAPv3. Ceci vous permet de consulter et de récupérer les informations utilisateur OpenSSO sans apporter de modification à votre base de données utilisateur existante. La structure OpenSSO intègre les données du plug-in du référentiel d'identités avec les données d'autres plug-ins OpenSSO afin de constituer une identité virtuelle pour chaque utilisateur. OpenSSO peut alors utiliser cette identité universelle dans les processus d'authentification et d'autorisation au sein de plusieurs référentiels d'identités. L'identité d'utilisateur virtuelle est détruite lorsque la session de l'utilisateur prend fin.

OpenSSO prend en charge les magasins de données suivants :

  • Active Directory

  • LDAPv3 générique

  • Sun Directory Server avec schéma OpenSSO

Pour plus d'informations sur la création d'un magasin de données et sur la configuration des plug-ins du référentiel, reportez-vous à la rubrique Création d'un magasin de données.
