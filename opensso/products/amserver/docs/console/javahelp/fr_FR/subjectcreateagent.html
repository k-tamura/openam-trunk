Création ou modification d'un agent
  1. Cliquez sur l'onglet Agents.
  2. Cliquez sur Nouveau.
  3. Renseignez les champs suivants :

    Name (Nom). saisissez le nom ou l' identité de l' agent. Il s'agit du nom que l'agent utilisera pour se connecter à Federated Access Manager. Les noms sur plusieurs octets ne sont pas acceptés.

    Password (Mot de passe). saisissez le mot de passe de l' agent. Ce mot de passe doit être différent du mot de passe utilisé par l'agent lors de l'authentification LDAP.

    Confirmez le mot de passe . Confirmez le mot de passe.

    Statut de l'unité. saisissez le statut du dispositif de l' agent. S'il est défini sur Actif, l'agent pourra s'authentifier auprès de Federated Access Manager et communiquer avec Access Manager. S'il est défini sur Inactif, l'agent ne pourra pas s'authentifier auprès de Federated Access Manager.

  4. Cliquez sur OK.
  5. Une fois que vous avez créé l'agent, vous pouvez également modifier les champs suivants :

    Description (Description). décrivez l' agent brièvement. Par exemple, vous pouvez entrer le nom de l'instance d'agent ou le nom de l'application qu'il protège.

    Valeur de clé d'agent. définissez les propriétés de l' agent avec une paire clé/agent. Cette propriété est utilisée par Federated Access Manager pour recevoir les demandes d'assertions d'informations d'authentification sur les utilisateurs par les agents. Actuellement, seule une propriété est valide et toutes les autres propriétés seront ignorées. Utilisez le format suivant :

    URLRacineAgent=http://nom_serveur:port/
