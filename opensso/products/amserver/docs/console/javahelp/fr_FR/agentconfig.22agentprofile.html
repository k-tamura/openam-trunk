Agent de stratégie 2.2

OpenSSO est compatible de manière ascendante avec Policy Agent 2.2. La suite Policy Agent 2.2 doit être configurée localement à partir du conteneur de déploiement sur lequel elle est installée. Par conséquent, à partir de la console OpenSSO, un nombre d'options Policy Agent 2.2 très limité peut être configuré.

Mot de passe

Le mot de passe a été défini lors de la création du profil d'agent. Cependant, vous pourrez changer de mot de passe à tout moment.

Confirmation de mot de passe

Le mot de passe a été confirmé lors de la création du profil d'agent. Si vous modifiez le mot de passe, vous devez confirmer votre modification.

Statut

L'option Active est sélectionnée lors de la création de l'agent. Ne choisissez Inactive que si vous voulez supprimer la protection assurée par l'agent.

Description

Description de l'agent, que vous pouvez ajoutez si vous le souhaitez.

Valeur de clé d'agent

Paramètre requis lors de l'activation de la connexion unique inter-domaine (CDSSO) et lors de la configuration du déploiement pour empêcher l'interception de cookies.

Cet attribut joue le rôle d'une clé dans une paire clé-valeur. Il permet à OpenSSO de recevoir les demandes d'assertions d'informations d'authentification sur les utilisateurs, émises par les agents. Seul un attribut est pris en compte dans cette paire clé-valeur. Tous les autres attributs sont ignorés. Utilisez le format suivant :

agentRootURL=protocol://hostname:port/

La saisie doit être exacte. Par exemple, la chaîne représentant la clé agentRootURL fait la distinction entre majuscules et minuscules.
