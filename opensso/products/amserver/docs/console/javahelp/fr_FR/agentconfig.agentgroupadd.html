Pour créer un nouveau groupe

Créez un groupe si vous souhaitez que les agents héritent de propriétés spécifiques du groupe. Les agents ne peuvent hériter leurs propriétés que de leur groupe. Par exemple, les agents Web peuvent hériter des propriétés d'un groupe d'agents Web.

  1. Cliquez sur l'onglet Contrôle d'accès.
  2. Cliquez sur le nom du domaine auquel le groupe appartiendra.
  3. Cliquez sur l'onglet Agents.
  4. Si nécessaire, sélectionnez l'onglet correspondant au type d'agent approprié.
  5. Dans la section Groupe, cliquez sur Nouveau.

    Dans le champ Nom, entrez le nom du nouveau groupe.

  6. Pour les agents Web et J2EE uniquement, saisissez l'URL du serveur OpenSSO.

    Par exemple, http://OpenSSO_EnterpriseHost.exemple.com:8080/OpenSSO .

  7. Cliquez sur Créer.

    La console crée le groupe d'agents et affiche de nouveau la page des agents, avec un lien pointant vers ce groupe.

    Pour effectuer d'autres tâches de configuration du groupe, cliquez sur ce lien afin d'afficher la page Modification de groupe d'agents.

    Les propriétés que vous pouvez définir pour configurer un groupe sont les mêmes que celles d'un agent, si ce n'est que les propriétés Groupe, Mot de passe et Confirmation de mot de passe ne sont pas disponibles au niveau du groupe.

    Remarque – En outre, sachez que certaines propriétés de groupe ont déjà des valeurs variables devant, dans la plupart des cas, ne pas être changées. Voici un exemple d'une de ces valeurs :

    @AGENT_PROTO@://@AGENT_HOST@:@AGENT_PORT@/amagent
