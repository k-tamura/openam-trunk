Attributs de liaison SOAP

Authentificateur de service Web

Cet attribut accepte comme valeur la classe d'implémentation de l'interface Authentificateur de service Web. Cette classe authentifie une demande et génère des données d'identité pour un consommateur de service Web.

Mécanismes d'authentification pris en charge

Cet attribut spécifie les mécanismes d'authentification pris en charge par le récepteur SOAP. Les mécanismes d'authentification permettent l'authentification des utilisateurs et assurent le chiffrement et l'intégrité des données. Par défaut, tous les mécanismes d'authentification disponibles sont sélectionnés. Si l'un d'entre eux n'est pas sélectionné et qu'un consommateur de services Web envoie une demande faisant appel à ce mécanisme, la demande est rejetée. Voici la liste des mécanismes d'authentification pris en charge :

  • urn:liberty:securité:2003-08:null:null

  • urn:liberty:securité:2003-08:null:X509

  • urn:liberty:securité:2003-08:null:SAML

  • urn:liberty:securité:2004-04:null:Bearer

  • urn:liberty:securité:2003-08:TLS:null

  • urn:liberty:securité:2003-08:TLS:X509

  • urn:liberty:securité:2003-08:TLS:SAML

  • urn:liberty:securité:2004-04:TLS:Bearer

  • urn:liberty:securité:2003-08:ClientTLS:null

  • urn:liberty:securité:2003-08:ClientTLS:X509

  • urn:liberty:securité:2003-08:ClientTLS:SAML

  • urn:liberty:securité:2004-04:ClientTLS:Bearer

N'appliquer que les fournisseurs connus

Si cette propriété est activée, elle applique l'en-tête ProviderID envoyé dans un message SOAP pour vérifier que le fournisseur existe dans le système. Si elle n'est pas activée, la demande est rejetée. Si cet attribut n'est pas activé, la vérification est ignorée.

Alias de certification de l'authentification du client SSL

Cette valeur est définie pendant l'installation. Alias de certificat client allant être utilisé dans la connexion SSL pour la liaison SOAP Liberty.

Limite horaire du message périmé

La valeur par défaut est 300 000. Détermine si un message est périmé et s'il n'est donc plus fiable. Si l'horodatage du message est antérieur à l'horodatage par le nombre de millisecondes spécifié, le message est considéré comme périmé.

Intervalle de nettoyage de cache d'ID de message

La valeur par défaut est 60 000. Spécifie le nombre de millisecondes devant s'écouler avant le début du nettoyage de cache. Chaque message est stocké dans un cache avec son propre ID pour éviter les messages dupliqués. Lorsque la durée actuelle d'un message moins la durée reçue dépasse la valeur thestaleTimeLimit, le message est supprimé du cache.

Acteurs SOAP pris en charge

La valeur par défaut est http://schemas.xmlsoap.org/soap/actor/next . Spécifie les acteurs SOAP pris en charge. Chaque acteur doit être séparé par une barre verticale (|).

Mappage de préfixe d'espace de noms

La valeur par défaut est :

=S=http://schemas.xmlsoap.org/soap/envelope/|sb=urn:liberty:sb:2003-08
|pp=urn:liberty:id-sis-pp:2003-08|ispp=http://www.sun.com/identity/
liberty/pp|is=urn:liberty:is:2003-08

Spécifie le mappage de préfixe d'espace de noms lors de la conversion d'un arbre de contenu JAXB vers un arbre DOM. La syntaxe est prefix=namespace|prefix=namespace|...

Liste des packages JAXB

Spécifie la liste de packages JAXB utilisée lors de la construction de JAXBContext . Chaque package doit être séparé par deux-points (:).

Version de service Web d'identité Liberty

Cette propriété détermine la structure Liberty ID-WSF lorsque la structure ne peut pas déterminer via le message entrant ou l'offre de ressources le moment où OpenSSO opère en tant que client du service Web. Les valeurs peuvent être 1.0 ou 1.1. La valeur par défaut est 1.1.
