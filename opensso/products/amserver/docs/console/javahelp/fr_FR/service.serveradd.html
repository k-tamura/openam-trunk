Pour créer une instance de serveur

Une entrée correspondant à chaque serveur est automatiquement créée dans la liste des serveurs lorsque OpenSSO Configurator est lancé pour configurer des serveurs. Normalement, ces opérations ne sont pas obligatoires.

  1. Connectez-vous à la console OpenSSO en tant qu'administrateur niveau supérieur.
  2. Cliquez sur l'onglet Configuration, puis sur Sites et serveurs.
  3. Cliquez sur Nouveau dans la liste des serveurs.
  4. Entrez le nom de domaine complet (FQDN) du serveur à ajouter, puis cliquez sur OK.

    Le FQDN doit avoir le format suivant : http (s)://hôte.domaine:port /uri.

  5. L'instance de serveur nouvellement créée apparaît dans la liste.
  6. Pour modifier le serveur, cliquez sur son nom. Vous pouvez configurer les attributs de configuration du serveur.

    Les paramètres de serveur par défaut constituent le groupe de valeurs par défaut des instances de serveur. Chaque instance de serveur doit comporter un jeu minimal de valeurs de propriétés et, selon votre déploiement, la plupart de ces valeurs peuvent être les mêmes pour toutes les instances de serveur. Ce paramètre vous permet de spécifier les propriétés de base en un endroit, sans devoir les changer pour chaque instance de serveur supplémentaire.

    Ces valeurs par défaut peuvent être écrasées. Pour ce faire, cliquez sur le bouton Paramètres d'héritage, situé en haut de la page de profil d'instance de serveur. Lorsque vous cliquez sur ce bouton, la console affiche une page sur laquelle vous pouvez sélectionner et désélectionner les valeurs dont hériter et les valeurs à écraser.
