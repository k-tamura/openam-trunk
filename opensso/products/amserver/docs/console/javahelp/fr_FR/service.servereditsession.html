Session

Les attributs de session vous permettent de configurer des informations de session pour un site et des instances de serveur supplémentaires.

Limites de session

Les attributs suivants définissent les limites de session de serveur :

Nombre maximum de sessions

La valeur par défaut est 5000. Spécifiez le nombre maximal de sessions simultanées autorisées. À la connexion, une erreur de nombre de sessions maximal est générée si la valeur de sessions simultanées maximale dépasse ce nombre.

Durée maximale de session non valide

La valeur par défaut est 10. Spécifie le nombre de minutes après lequel la session non valide est supprimée de la table des sessions si cette table est créée et si l'utilisateur ne se connecte pas. Cette valeur doit toujours être supérieure à la valeur de temporisation indiquée dans le fichier de propriétés du module d'authentification.

Délai de purge de session

La valeur par défaut est 60. Spécifie, en minutes, le délai de retard de l'opération de session de purge. Après expiration d'une session, il s'agit d'une période de prolongation pendant laquelle la session continue à résider sur le serveur de session. Cette propriété est utilisée par l'application cliente pour vérifier si la session a expiré via les API SSO. À l'issue de cette période de prolongation, la session est arrêtée. Si l'utilisateur se déconnecte ou si la session est explicitement arrêtée par un composant OpenSSO, elle n'est pas maintenue pendant la période de prolongation. Pendant la période de prolongation, la session est dans l'état NON VALIDE.

Statistiques

Les attributs suivants définissent la configuration des statistiques :

Intervalle enregistrement

La valeur par défaut est 60. Spécifie le délai en minutes séparant les journalisations des statistiques. La valeur minimale est de 5 secondes pour éviter la saturation de la CPU. OpenSSO considère qu'une valeur inférieure à 5 secondes est égale à 5 secondes.

État

La valeur par défaut est file. Spécifie l'emplacement du journal des statistiques. Les valeurs possibles sont les suivantes :

  • off — Aucune statistique n'est consignée.

  • file — Les statistiques sont écrites dans un fichier situé dans le répertoire spécifié.

  • console — Les statistiques sont écrites dans les fichiers journaux du serveur Web.

Répertoire

Cette valeur est définie pendant l'installation. Exemple : base OpenSSO/URI-serveur/stats. Spécifie le répertoire dans lequel les fichiers de débogage sont créés.

Activer la recherche d'hôte

La valeur par défaut est false. Active ou désactive la recherche d'hôte pendant la journalisation de session.

Notification

Les attributs suivants définissent la configuration de notification :

Taille de pool de notification

La valeur par défaut est 10. Définit la taille du pool en spécifiant le nombre total de threads.

Seuil de pool de threads de notification

La valeur par défaut est 100. Spécifie la longueur maximale des files d'attente de tâches. Lorsqu'une tâche de notification est reçue, elle est envoyée à la file d'attente des tâches pour être traitée. Si la file d'attente atteint la longueur maximale, les nouvelles demandes entrantes sont rejetées avec un message ThreadPoolException jusqu'à ce qu'une tâche disparaisse de la file d'attente.

Validation

Les attributs suivants définissent la configuration de validation :

Comparaison DN client insensible à la casse

La valeur par défaut est true. Compare le DN d'agent. Si la valeur est false, la comparaison est sensible à la casse.
