Création d'une chaîne d'authentification

  1. Cliquez sur le nom du domaine auquel vous souhaitez ajouter une nouvelle chaîne d'authentification.
  2. Sélectionnez l'onglet Authentification.
  3. Cliquez sur Nouveau dans la liste Enchaînement d'authentification.
  4. Entrez le nom de la chaîne d'authentification.
  5. Cliquez sur OK.
  6. Cliquez sur Ajouter pour définir l'instance du module d'authentification que vous souhaitez inclure dans la chaîne. Pour ce faire, sélectionnez le nom de l'instance de module dans la liste des instances. Les noms d'instance de module affichés dans cette liste sont créés dans l'attribut Instances de module.
  7. Sélectionnez les critères de la chaîne. Ces indicateurs définissent des critères d'application destinés au module d'authentification pour lequel ils sont définis. Le processus d'application suit une hiérarchie. Obligatoire est le plus élevé et Facultatif est le plus bas :
    Requis

    L'authentification de l'instance de module doit réussir. En cas de réussite, l'authentification se poursuit avec la liste Enchaînement d'authentification. En cas d'échec, le contrôle est immédiatement rendu à l'application (l'authentification ne se poursuit pas avec la liste Enchaînement d'authentification).

    Requis

    L'authentification vis-à-vis de ce module doit réussir. Si l'un des modules de la chaîne échoue, l'ensemble de la chaîne d'authentification sera en échec. Toutefois, quel que soit le résultat pour un module (réussite ou échec), le contrôle est transféré au module suivant de la chaîne.

    Suffisant

    L'authentification de l'instance de module n'est pas obligatoire. En cas de réussite, le contrôle est immédiatement rendu à l'application (l'authentification ne se poursuit pas avec la liste des instances de module). En cas d'échec, l'authentification se poursuit avec la liste Enchaînement d'authentification.

    Facultatif

    L'authentification de l'instance de module n'est pas obligatoire. Quel que soit le résultat de l'authentification (réussite ou échec), l'authentification se poursuit avec la liste Enchaînement d'authentification.

  8. Entrez les options pour la chaîne. Ceci permet d'entrer des options supplémentaires pour le module comme une paire clé=valeur. Séparez les options par un espace.
  9. Définissez les attributs suivants :
    URL de connexion réussie

    Spécifie l'URL vers lequel l'utilisateur est redirigé en cas d'authentification réussie.

    URL d'échec de connexion

    Spécifie l'URL vers lequel l'utilisateur est redirigé en cas d'échec de l'authentification.

    Classe de post-traitement de l'authentification

    Définit le nom de la classe Java utilisée pour personnaliser le processus de post-authentification après la réussite ou l'échec d'une connexion.

  10. Cliquez sur Enregistrer.
