要求ハンドラを設定する

RequestHandler インタフェースは、SOAP バインドサービスを使用している Liberty ベースの各アイデンティティーサービスによって、サーバー側に実装される必要があります。「要求ハンドラリスト」属性は、要求ハンドラを実装している Web サービスの実装クラスに関する情報を格納します。要求ハンドラリストには、キーと値のペアを含むエントリがパイプ文字で区切られて表示されます。たとえば、key=disco|class=com.example.identity.liberty.ws.disco.DiscoveryService のように表示されます。

  1. 「新規」をクリックして「新規要求ハンドラ」属性を表示するか、設定済みのキー値をクリックして既存の属性を変更します。
  2. 次の情報に基づいて属性の値を入力します。
    キー

    この必須パラメータは、SOAP エンドポイントに対する URI パスの最後の部分です。OpenSSO の SOAP エンドポイントは、SOAP 受信者サーブレットです。SOAP Receiver への URI は protocol://host:port/deploy-uri/Liberty/key です。キーとして disco を定義する場合、対応するディスカバリサービスの SOAP エンドポイントへの URI パスは、protocol://host:port/amserver/Liberty/disco のようになります。SOAP Receiver への接続時には、さまざまなサービスクライアントがそれぞれ別のキーを使用します。

    クラス

    この必須パラメータは、特定のアイデンティティーサービスの要求ハンドラ実装クラスの名前を指定します。次に例を示します。class=com.example.identity.liberty.ws.disco.DiscoveryService

    SOAP アクション

    このパラメータは、オプションの SOAP 固有の URI を指定します。

  3. 「了解」をクリックして要求ハンドラの設定を完了します。
  4. SOAP バインドサービスページで「保存」をクリックして、サービスの設定を完了します。
