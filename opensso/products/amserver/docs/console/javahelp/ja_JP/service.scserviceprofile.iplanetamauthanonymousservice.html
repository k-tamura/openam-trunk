匿名

このモジュールタイプにより、ユーザーは証明情報を指定せずにログインできます。だれでもパスワードを入力せずに匿名でログインできるようにするために、匿名ユーザーを作成できます。匿名接続は通常、匿名ユーザーがサーバーに対するアクセスを制限されるように OpenSSO 管理者によってカスタマイズされます。匿名認証属性はレルム属性です。属性は次のとおりです。

  • 「有効な匿名ユーザー」

  • 「デフォルトの匿名ユーザー名」

  • 「大文字と小文字を区別するユーザー ID」

  • 「認証レベル」

有効な匿名ユーザー

証明情報を指定しないでログインする権限のあるユーザー ID のリストが含まれています。ユーザーのログイン名がこのリストのユーザー ID に一致した場合は、アクセスが許可され、指定したユーザー ID にセッションが割り当てられます。

このリストが空の場合、次のデフォルトモジュールインスタンスログイン URL にアクセスすると、デフォルトの匿名ユーザー名として認証されます。

protocol://server_host.server_domain:server_port/server_deploy_uri/UI/Login?module=Anonymous&org=org_name

このリストが空でない場合、デフォルトモジュールインスタンスログイン URL (上記と同じ) にログインすると、ユーザーは有効な匿名ユーザー名を入力するよう要求されます。このリストが空でない場合は、ユーザーは次の URL にアクセスしてログインページを表示せずに、ログインすることができます。

protocol://server_host.server_domain:server_port/server_deploy_uri/UI/Login?module=Anonymous&org=org_name&IDToken1=<valid Anonymous username>

デフォルトの匿名ユーザー名

「有効な匿名ユーザーリスト」が空白で、次のデフォルトモジュールインスタンスログイン URL がアクセスされた場合に、セッションが割り当てられるユーザー ID を定義します。

protocol://server_host.server_domain:server_port/server_deploy_uri/UI/Login?module=Anonymous&org=org_name

デフォルト値は anonymous です。匿名ユーザーはレルムにも作成する必要があります。

注 – 「有効な匿名ユーザーリスト」が空白でない場合は、「デフォルトの匿名ユーザー名」で定義されたユーザーを使用して、ログインページにアクセスせずにログインすることができます。これは、次の URL にアクセスすることで実行できます。

protocol://server_host.server_domain:server_port/server_deploy_uri/UI/Login?module=Anonymous&org=org_name&IDToken1= DefaultAnonymous User Name

大文字と小文字を区別するユーザー ID

有効な場合、このオプションでユーザー ID の大文字と小文字を区別できます。デフォルトでは、この属性は無効です。

認証レベル

認証レベルは認証方法ごとに個別に設定します。この値は、認証メカニズムの信頼度を示します。ユーザーが認証を受けると、この値がセッションの SSO トークンに格納されます。ユーザーがアクセスするアプリケーションに SSO トークンが提供されると、そのアプリケーションは格納されている値を使用して、ユーザーにアクセスを許可するのに十分なレベルかどうかを判別します。SSO トークンに格納されている認証レベルが必要な最小値に満たない場合、アプリケーションはユーザーにより高い認証レベルのサービスで認証を再度受けるよう要求することがあります。デフォルト値は 0 です。

注 – 認証レベルの指定がない場合、SSO トークンはコア認証属性の 「デフォルト認証レベル」で指定した値を格納します。
