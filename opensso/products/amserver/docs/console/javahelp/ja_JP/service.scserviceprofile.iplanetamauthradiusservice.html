RADIUS

このモジュールにより、外部 RADIUS (Remote Authentication Dial-In User Service) サーバーを使用した認証が可能になります。RADIUS 認証属性はレルム属性です。属性は次のとおりです。

  • 「サーバー 1」

  • 「サーバー 2」

  • 「共有シークレット」

  • 「共有シークレット (確認)」

  • 「ポート番号」

  • 「タイムアウト」

  • 「認証レベル」

サーバー 1

プライマリ RADIUS サーバーの IP アドレスまたは完全修飾ホスト名を表示します。デフォルト IP アドレスは 127.0.0.1 です。フィールドでは任意の有効な IP アドレスまたはホスト名が認識されます。複数のエントリの場合は、次の構文のようにローカルサーバー名をプレフィックスとして付ける必要があります。

local_servername|ip_address local_servername2|ip_address ...

サーバー 2

セカンダリ RADIUS サーバーの IP アドレスまたは完全修飾ドメイン名 (FQDN) を表示します。これはフェイルオーバーサーバーで、プライマリサーバーが通信できない場合に通信するサーバーです。デフォルト IP アドレスは 127.0.0.1 です。複数のエントリには、次の構文のようにローカルサーバー名をプレフィックスとして付ける必要があります。

local_servername|ip_address local_servername2|ip_address ...

共有シークレット

RADIUS 認証の共有シークレットを保持します。共有シークレットは、注意深く選んだパスワードと同じレベルにする必要があります。このフィールドのデフォルト値はありません。

共有シークレット (確認)

RADIUS 認証の共有シークレットを確認します。

ポート番号

RADIUS サーバーが待機するポートを指定します。デフォルト値は 1645 です。

タイムアウト

RADIUS サーバーがタイムアウトするまで応答を待つ時間間隔を秒単位で指定します。デフォルト値は 3 秒です。どのような秒数のタイムアウトを指定しても認識されます。

認証レベル

認証レベルは認証方法ごとに個別に設定します。この値は、認証メカニズムの信頼度を示します。ユーザーが認証を受けると、この値がセッションの SSO トークンに格納されます。ユーザーがアクセスするアプリケーションに SSO トークンが提供されると、そのアプリケーションは格納されている値を使用して、ユーザーにアクセスを許可するのに十分なレベルかどうかを判別します。SSO トークンに格納されている認証レベルが必要な最小値に満たない場合、アプリケーションはユーザーにより高い認証レベルのサービスで認証を再度受けるよう要求することがあります。デフォルト値は 0 です。

注 – 認証レベルの指定がない場合、SSO トークンはコア認証属性の 「デフォルト認証レベル」で指定した値を格納します。
