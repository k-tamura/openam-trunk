ディスカバリサービス

アイデンティティーデータにアクセスするための最初の手順は、情報のある場所を見極めることです。たとえば、主体のクレジットカード情報を保持しているアイデンティティーサービスや、主体のカレンダサービスを格納しているサーバーなどを判断します。通常、ネットワーク上には、ほかのエンティティーがアイデンティティーデータに対して動作を行えるようにするサービスが 1 つ以上あります。クライアントにディスカバリサービスが必要な理由は、クライアントがこれらのサービスを追跡すること、または、どのサービスが信頼できるかをクライアントが認識していることが想定されていないためです。「ディスカバリサービス」は、基本的にはディスカバリリソース用の Web サービスインタフェースです。「ディスカバリリソース」とは、リソースオファリングのレジストリの一種です。「リソースオファリング」では、アイデンティティーデータと、そのデータへのアクセスを提供するサービスインスタンスとの間の関連付けが定義されます。「リソース ID」はディスカバリサービスに登録された一意のリソース識別子 (URI) で、特定のディスカバリリソースを指します。クライアントがある種類のデータの要求を送信する場合、要求にはリソース ID が含まれており、OpenSSO ディスカバリサービスはこれを使用して、要求された属性の Web サービスプロバイダを見つけます。ディスカバリサービスは、データを見つけるために必要な情報を含むリソースオファリングを返します。

注 – アクセスが発生するためには、各アイデンティティー主体の代わりに、Liberty 個人プロファイルサービスのホスティングプロバイダをディスカバリサービスに登録する必要があります。

実装環境には、次のディスカバリサービスのグローバル属性を設定できます。

  • 「プロバイダ ID」

  • 「サポートされている認証メカニズム」

  • 「サポートされているディレクティブ」

  • 「ディスカバリ検索のポリシー評価」

  • 「ディスカバリ更新のポリシー評価」

  • 「オーサライザプラグインクラス:」

  • 「エントリハンドラプラグインクラス:」

  • 「リソース ID マッパープラグイン用のクラス」

  • 「応答メッセージの認証」

  • 「ブートストラップ用のセッションコンテキストステートメント」

  • 「ブートストラップ用のセッションコンテキスト内での NameIdentifier の暗号化」

  • 「暗黙のリソース」

  • 「ブートストラップのリソースオファリング」

ディスカバリサービスの設定に関連している作業は、次のとおりです。

  • 「ブートストラップのリソースオファリングを設定する」

  • 「サービス記述を設定する」
