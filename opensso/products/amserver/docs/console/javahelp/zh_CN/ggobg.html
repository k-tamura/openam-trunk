创建新组

如有必要，在 Federated Access Manager 管理控制台中执行该任务。该任务适用于 Web 代理组和 J2EE 代理组。

如果要使代理继承组的特定属性，则创建一个组。J2EE 代理可以继承 J2EE 代理组的属性，而 Web 代理可以继承 Web 代理组的属性。

  1. 单击“配置”选项卡下的“代理”。
  2. 如有必要，选择适当代理类型的选项卡。
    • Web 代理：

      "Web" 选项卡适用于 Web 代理且为默认选项。无需再次选择。

    • J2EE 代理：

      "J2EE" 选项卡适用于 J2EE 代理。如果适用，单击该选项卡。

  3. 在“组”部分中单击“新建”。

    在“名称”字段中输入新组的名称。

  4. 在“服务器 URL”字段中，输入 Federated Access Manager 服务器 URL。
  5. 单击“创建”。

    控制台创建代理组，并再次显示代理（WebAgent 或 J2EEAgent）页面，其中包含指向该组的链接。

    要对组进行其他配置，单击该链接显示“编辑”代理组页面。

    您设置为配置组的属性与用于个体代理的属性相同，仅有的不同之处是在该组级别下不能使用“组”、“密码”、“密码确认”属性。

    注 – 此外，要注意到一些组属性已经分配了在大多数情况下不应更改的变量值。此类值的一个示例如下：

    @AGENT_PROTO@://@AGENT_HOST@:@AGENT_PORT@/amagent
