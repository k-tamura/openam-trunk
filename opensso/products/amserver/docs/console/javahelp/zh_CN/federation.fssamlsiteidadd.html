配置站点标识符

此属性定义了由装有 OpenSSO 的服务器托管的任何站点。在安装过程中会为主机定义默认值和自动生成的站点 ID。可能有多个条目。例如，负载平衡或共享同一个 Directory Server 的多个 OpenSSO 实例都需要进行定义。起点是联合界面下的 "SAML" 屏幕上的“站点标识符”属性。站点 ID 在“服务器和站点配置”屏幕中定义。有关详细信息，参见服务器和站点。

  1. 单击“新建”以添加新的站点标识符；或单击已配置的站点标识符的名称，对其配置文件进行修改。

    随即显示“站点标识符”属性。

  2. 基于以下信息，为“站点标识符”属性提供值：
    实例 ID

    此属性的值为 protocol://host:port。

    如果要为 SSL 配置 SAML（同时在源站点和目标站点中），请确保此处定义的协议为 https//。

    站点 ID

    站点 ID 是为每个站点生成的标识符（尽管该值对于负载平衡器背后的多个服务器相同）。com.sun.identity.saml.common 软件包中有一个类，如果需要，可以使用该类手动生成此标识符。在命令行中键入以下命令：

    % java -classpath FM-classpath com.sun.identity.saml.common.SAMLSiteID protocol://host:port
    发布者名称

    此属性的默认值为 host:port，但它可以是任何 URI。

  3. 单击“确定”完成“站点标识符”的配置操作。
  4. 单击“本地站点属性”页面上的“保存”以完成 SAML 的配置操作。
