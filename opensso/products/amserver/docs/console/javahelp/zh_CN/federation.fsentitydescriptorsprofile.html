修改提供者实体或联合提供者实体的常规属性

创建实体之后，您可以编辑其配置文件。这可能包括修改属性、添加身份认证提供者和服务提供者，或配置联合提供者。编辑实体配置文件的“常规”属性时，可能需要修改已定义的公共属性、添加联系人，或提供可选的组织信息。可按以下过程编辑“常规”属性。起点是联合下的“实体”屏幕。

  1. 选择已配置实体的名称以修改其配置文件。

    默认情况下，将显示实体配置文件的“常规”属性页面。如果当前不处于“常规”属性页面，可从“查看”菜单中选择“常规”。

  2. 编辑任何实体公共属性的值。
    实体名称

    该属性的静态值是您创建实体时提供的名称。

    类型

    该属性的静态值为实体类型（“提供者”或“联合提供者”）。

    说明

    该属性的值是您创建实体时提供的说明。可以修改原来输入的说明，如果不存在，则可添加一条说明。

  3. 提供“实体联系人配置文件”属性的值。

    “实体联系人配置文件”属性提供实体联系人方面的信息。这些属性都是可选的。

    名字

    键入实体联系人的名。

    姓氏

    键入实体联系人的姓。

    类型

    从下拉式菜单中选择联系人类型：

    • 管理

    • 记帐

    • 技术

    • 其他

    公司

    键入雇用该人员的公司的名称。

    Liberty 主体 ID

    键入指向联系人个人信息配置文件实例的统一资源标识符 (Uniform Resource Identifier, URI)。

    电子邮件

    键入联系人的一个或多个电子邮件地址，然后单击“添加”。

    电话号码

    键入联系人的一个或多个电话号码，然后单击“添加”。

  4. 提供“组织配置文件”属性的值。

    “组织配置文件”属性提供与主体交互时可能需要的基本信息。这些属性都是可选的。

    注 – 如果“名称”属性包含值，则需要向“显示名称”属性和 "URL" 属性添加值。

    名称

    输入实体组织的合法名称并单击“添加”。按 locale|organization-name 格式定义该值。例如，en|example.com。

    显示名称

    输入适于向主体显示的实体组织名称，然后单击“添加”。按 locale|organization-display-name 格式定义该值。例如，en|example.com。

    URL

    输入组织的 URL 并单击“添加”。按 locale|organization-URL 格式定义该值。例如，en|http://www.example.com。

  5. 单击“保存”完成配置，或者继续定义身份认证提供者、服务提供者或联合提供者属性的值。

    如果要继续配置，请参见：

    • 向提供者实体添加身份认证提供者，定义身份认证提供者属性的值。
    • 向提供者实体添加服务提供者，定义服务提供者属性的值。
    • 向联合提供者实体添加成员，以成员填充联合提供者实体。
