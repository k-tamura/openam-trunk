Q:OpenSSO とは何ですか?

OpenSSO と称されている Open Web Single Sign-On プロジェクトは、 Sun Microsystems, Inc. より提供されていたアイデンティティと連携(フェデレーション)に関する 2 つの商用製品である Sun JavaTM System Access Manager と Sun Java System Federation Manager のソースコードをベースにしたオープンな開発プロジェクトです。 OpenSSO のゴールはパブリックドメインにおけるアイデンティティサービスインフラに対する拡張可能な基盤を提供したり、Web やアプリケーションサーバーでホストされる Web アプリケーションに対するシングルサインオン(SSO) の実現を手助けすることです。 OpenSSO のコードベースは Sun Microsystems からリリースされるアイデンティティおよび連携関連の次期商用製品の基盤となります。

Q: Sun Java System Access Manager とは何ですか?

Sun Java System Access Manager は、Sun Microsystems がリリースしている商用製品で、 OpenSSO のコードのベースとなっているものです。

Q: Sun Java System Federation Manager とは何ですか?

Sun Java System Federation Manager は、Sun Microsystems がリリースしている商用製品で、OpenSSO のコードのベースとなっているものです。

Q: OpenSSO のコードベースは Access Manager や Federation Manager のものと異なりますか?

コードベースは同じです。 Access Manager や Federation Manager をリリースした後に、Sun はコードを寄付し、 OpenSSO オープンソースプロジェクトを立ち上げました。 それ以降、Sun のエンジニアと OpenSSO コミュニティがオープンな環境でコードを開発し、ナイトリービルドを作り、最新の機能をテストしたり使用できるように、定期的に安定したダウンロード用ビルドを作成しています。 2009年1月現在、Sun OpenSSO Enterprise 8.0 が OpenSSO の最新の商用製品としてリリースされています。詳細は、こちらを参照してください。

Q: Sun Microsystems はどのようにして OpenSSO をサポートしますか?

2008年7月23日より、Sun Java System Access Manager または OpenSSO のサポートを受けるには、以下のことが必要です:

  1. Sun Java System Access Manager の perpetual license、または Sun Identity Management Suite か Java Enterprise System の subscription を購入する
  2. Sun Java System Access Manager または、OpenSSO Express ビルド または Sun OpenSSO Enterprise 版をダウンロードする
その他は必要ありません。お客様は、OpenSSO のサポートや法的補償を受けるのに追加のライセンスやサポート契約を購入する必要はありません。 Sun Java System Federation Manager は、OpenSSO を介してはサポートされない製品であることに注意してください。

Q: OpenSSO Express ビルドとは何ですか?

ナイトリービルドの中で、最新の 2 つの安定版ビルドを OpenSSO Express ビルドと呼んでいます。OpenSSO Express ビルドは、 Sun Quality Assurance (QA) engineering チームによるマニュアルテストと自動化テストを経てリリースされています。 Sun Customer Support チャネルから問題をレポートするには、OpenSSO Enterprise 版か、その時点でサポートされている OpenSSO Express ビルドを配備している必要があります。

Q: どれくらいの間隔で新しい OpenSSO Express ビルドがリリースされますか?

新しい OpenSSO Express ビルドはだいたい 3 ヶ月に 1 回のペースでリリースされています。

Q: OpenSSO Community ビルドとは何ですか?

OpenSSO のサイトにあるナイトリービルドを Community ビルド と呼んでいます。Community ビルドには常にその時点での最新のコードが含まれていますが、 Sun Quality Assurance (QA) engineering チームによるテストは行われていません。

Q: OpenSSO Community ビルドはサポートされますか?

OpenSSO Community ビルドのサポートは、OpenSSO コミュニティ上のみで行われ、IRC や メーリングリスト、メンバーのブログ、OpenSSO wiki、 OpenSSO web サイトにアップされているドキュメントマニュアルや記事を通じて行われます。

Q: 最新の OpenSSO Express ビルドがリリースされたら、それにアップグレードする必要がありますか?

最新ビルドにアップグレードしなくてもかまいません。必要な限り、現在のビルドをそのまま使っていただいて、その後、サポート契約が有効である間に、いつでも構いませんので、より新しいビルドや商用リリース版にアップグレードしてください。 ただし、もしあなたが何か問題を見つけて、サポートチームに連絡を入れた場合、サポートチームから最初にお願いされるのは、最新のビルドにアップグレードすること、になると思いますのでご承知おきください。

Q: OpenSSO Express ビルドを使用している場合、サポートを受けられますか?

まず最初に、問題をレポートする時点で、現在使用されているビルドが OpenSSO Express ビルドの一覧に含まれているかどうかを確認してください。 もし含まれていれば、通常の Sun Customer Support チャネルを通じてサポートを受けられます。 もし含まれていなければ、現在お使いのビルドは古いものということになりますので、サポートにコールする前に、お使いの環境を現在サポートされている Express ビルドか、商用リリース(Enterprise 版)にアップグレードして、その問題が新しい環境でも再現するかを確認していただく必要があります。 課題トラッカーを使って、現在オープンになっている、または過去にオープンされていて現在解決済みであるバグを検索することができます。データベースの検索はどなたでも可能です。

Q: OpenSSO Express ビルドは、リリースされる前にどのようなテストが行われていますか?

Sun QA engineering チームにより広範囲に渡っての、マニュアルテストおよび自動化テストが行われています。また、サードパーティ製品との 相互運用(interoperability)テストも行われています。
