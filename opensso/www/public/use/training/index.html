Free Introductory Training Offer: Early Access Downloadable On Demand Labs

The OpenSSO Deployment early access course includes a series of five downloadable, self-paced labs that take you through a complex OpenSSO deployment. You deploy two Apache Tomcat servers, SSL-enable them, install a software load balancer, install OpenSSO into the environment, and configure OpenSSO for session failover. Then you install an example web server and an example application server, and install Policy Agent software to see how OpenSSO protects web sites and JavaTM 2 Platform, Enterprise Edition (J2EETM) applications.
This course uses OpenSSO Build 4.5, which provides identical functionality to OpenSSO Express Build 5. Other deployment components include Apache Tomcat version 6.0.14, Sun Java System Web Server version 7.0, and GlassFishTM application server version 2.
For this training, you need a system that meets the following requirements:
  • x86-based.
  • Runs the Windows, Mac O/S, or Linux operating system.
  • A minimum of 2 GB memory. (More is better.)
  • 30 GB of free space on a disk drive.
Get Started: First, sign up for an account at My Sun if you do not already have one. (Sun employees can use their Sun e-mail address and LDAP password to log in.) Then access the course from Sun Learning's lab portal.
Get Help While Taking the Training: Since this training is available at no cost to you, the Ask the Expert feature of Sun Learning's lab portal is inactive for this OpenSSO Deployment course. So use the OpenSSO community to get help instead. E-mail the mailing list, post to the forum, or use the IIRC channel.

Sun OpenSSO Enterprise 8.0: Deployment Essentials - Classroom Course

The Sun OpenSSO Enterprise 8.0: Deployment Essentials course is a 5-day classroom course that provides students with a description of the role of Sun OpenSSO Enterprise (OpenSSO) in an identity management solution. Students learn to deploy, configure, manage, and administer OpenSSO.
Upon completion of this course, students should be able to:
  • Define the role of OpenSSO in the enterprise computing environment, and list the features and functions of OpenSSO that address key business challenges
  • Download the OpenSSO source code and track OpenSSO issues
  • Deploy a multi-instance configuration with session failoveri
  • Install and configure Policy Agent 3.0 software
  • Configure OpenSSO using the console and the command line
  • Activate and configure audit logging and debugging features in OpenSSO
  • Configure OpenSSO realms and data stores
  • Configure and customize authentication and single sign-on
  • Configure authorization
  • Customize an OpenSSO instance
  • Remove OpenSSO components from a system
  • Deploy a federated identity configuration, including a fedlet
